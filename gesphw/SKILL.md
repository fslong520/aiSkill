# GESP作业生成器

## 适用场景

当用户要求：
- 生成某知识点的作业
- 制作练习卷/测验卷
- 出题
- 为某班级制作作业卷子

## 作业结构（符合GESP考试标准）

| 题型     | 题数 | 每题分值 | 总分    |
|----------|------|----------|---------|
| 选择题   | 10   | 10       | 100     |
| 判断题   | 5    | 10       | 50      |
| 编程题   | 6    | 100      | 600     |
| **合计** | 21   | -        | **750** |

## 文件命名规范

```
班级名/日期 - 知识点.md              # 作业题目
班级名/日期 - 知识点 - 答案.md        # 答案
```

**示例**：
- `庞桢扬班/4月 2 日 - 二进制与进制转换.md`
- `庞桢扬班/4月 2 日 - 二进制与进制转换 - 答案.md`

## 生成流程

### 1. 收集信息

确认以下信息（用户未明确则推断）：
- **班级名**：从上下文推断
- **日期**：从文件名推断
- **知识点**：**用户指定**，用户说什么就覆盖什么
- **排除内容**：用户明确不要的内容（如不要位运算、不要指针等）

### 2. 设计 Front Matter

**必须包含以下字段**：
```yaml
---
title: "日期 作业 - 故事主题"
description: 知识点1、知识点2、知识点3...（列出本卷涉及的具体知识点）
---
```

**示例**：
```yaml
---
title: "4 月 11 日作业 - 勇者的背包"
description: 变量、循环、分支、数组入门练习
---
```

### 3. 生成题目

**故事融入方式**：
- **不使用单独的故事背景章节**
- 直接在作业标题下方用简短的引语/情境描述引入（3-5行）
- 使用引用块 `>` 突出情境设定
- 每道题的标题要贴合故事主题（如【武器升级】【购买药水】）

**选择题要求**：
- 使用二级标题：`## 第 1 题【关卡名】(10分)`
- 每题包含4个选项（A/B/C/D）
- 选项之间空一行
- 正确选项随机分布
- 题目描述融入故事情境

**判断题要求**：
- 使用二级标题：`## 第 11 题【关卡名】(10分)`
- T表示正确，F表示错误
- 题目陈述要有一定迷惑性

### 4. 生成答案

**答案文件格式**：
```
1. X
2. X
3. X
...
15. X
16. cpp
17. cpp
```

（选择题和判断题用字母/T/F，编程题用 `cpp`）

### 5. 答案分布原则

**选择题**：
- A/B/C/D 四个选项都要出现
- 正确答案分布要均匀（各2-3个）
- 错误选项要有一定迷惑性

**判断题**：
- **禁止固定比例**！如3T2F、2T3F等固定模式
- 每套题单独设计，不能有规律可循
- 可以偶尔全部正确或全部错误（但要配合选择题整体难度）
- **核心原则**：学生无法通过猜测规律得分

### 6. 设计故事主题

**故事设计要求**：
- 根据知识点设计一个简洁的故事主题（如"勇者的背包""外星基地"等）
- 在作业标题下方用3-5行引语设置情境，使用引用块 `>`
- 引语要自然引出题目内容，有代入感
- **不要写长篇的故事背景章节**

**题目故事化要求**：
- 每道题的标题要贴合故事（如【武器升级】【购买药水】【背包格子】）
- 题目描述以故事角色的视角展开
- 编程题直接给出任务描述，**不额外包装故事背景段落**
- 让整套试卷有统一的故事情境但不冗长

### 7. 创建文件

使用 `create_file` 工具创建两个文件：
1. 作业题目文件
2. 答案文件

## 题目内容设计原则

### 知识点覆盖

- **知识点来源**：完全由用户指定，用户说什么知识点就覆盖什么
- 常见知识点：C++基础语法、循环结构、数组、字符串、结构体、排序算法、贪心算法、位运算、二进制、进制转换等
- 如果用户没有明确知识点，从上下文推断（查看最近的PPT或教学文档）

### 选择题设计

- 每题包含4个选项（A/B/C/D）
- 覆盖用户指定知识点的核心概念和应用
- 错误选项要有迷惑性，像真实易错点
- **故事化**：题目描述融入故事情境，成为故事中的一个关卡或挑战

### 判断题设计

- T表示正确，F表示错误
- 题目陈述要有一定迷惑性
- 可以考察概念辨析、常见误区等
- **故事化**：题目描述融入故事情境，成为故事中需要判断真假的线索

### 编程题设计

从 OJ 系统选取或创建与知识点匹配的编程题：
1. 选择6道合适的编程题
2. 每个编程题使用格式：`### 序号、<a href="题目链接" target="_blank">题目名称</a>(100分)`
3. **直接给出题目描述**，不额外包装故事背景段落
4. 题目标题要贴合故事主题（如"装备升级""勇者属性计算"）
5. 题目描述中可以融入故事情境，但保持简洁

**输入格式**

**样例输入**  
```

```

**样例输出**  
```
80
```

初始50，升级3次，每次+10，最终 = 50 + 10×3 = 80

## 注意事项

1. **Front Matter**：必须包含 title 和 description，description 列出具体知识点
2. **故事引入**：使用简短引语（3-5行）+ 引用块，不要写长篇故事背景章节
3. **标题层级**：选择题/判断题使用二级标题 `##`，编程题使用三级标题 `###`
4. **答案分布**：判断题不能有固定模式（如3T2F），答案要随机分布，避免学生猜测规律
5. **知识点定制**：严格按照用户指定的知识点出题
6. **排除内容**：用户明确不要的内容不要出现（如位运算）
7. **文件路径**：先确认班级目录存在，再创建文件
8. **编程题格式**：使用 `### 序号、<链接>题目名(100分)` 格式，直接给题目描述
