# 模块 05 - 类人检索策略

## 核心算法

人类记忆不是数据库查询，而是多维度加权的结果:

### 综合得分公式

```
最终得分 = 0.40 × 语义相似度
         + 0.15 × 情绪权重
         + 0.20 × 近因效应
         + 0.25 × (0.6 + 频率强化)
```

### 各分量计算

| 分量 | 权重 | 计算方式 | 说明 |
|------|------|----------|------|
| **语义相似度** | 40% | Chroma cosine 向量搜索 | AI 理解的意义匹配度 |
| **情绪权重** | 15% | 🔴极1.0 🟠高0.8 🟡中0.5 🟢低0.2 | 情绪越强烈，越容易忆起 |
| **近因效应** | 20% | exp(-ln(2) × 天数 / 30) | 半衰期30天，30天后回忆概率减半 |
| **频率强化** | 25% | 0.6 + log₂(f+1)×0.1 + log₂(r+1)×0.05 | f=出现频次, r=被回忆次数 |

## 检索模式

### 1. 基础语义检索

```bash
python3 scripts/memory_core.py recall "Python"
```

### 2. 联想扩散 (关键!)

```bash
python3 scripts/memory_core.py recall "Python" --expand
```

执行流程（两阶段扩散）:

**阶段一：关系链扩散**
1. 先做语义检索，获取 top-10 结果
2. 对 top-3 结果，通过 `relationships` 集合查直接关联关系（store 时自动建链）
3. 关系链上的新记忆以较低权重加入结果集

**阶段二：语义二次检索**
4. 取 top-2 结果之内容/关键字，以此为新查询再做语义检索
5. 发现与原始结果不重叠者，即为"涌现"
6. 最终统一按综合得分排序
7. 所有扩散结果标记为 `[关联]`

> **实现机制**：每条记忆在 `store` 时自动搜索语义相似(p>0.50)的已有记忆，
> 写入 `relationships` 集合并交叉融合关键字，使后续检索覆盖更广。

### 3. 情绪锚定检索

只检索高情绪记忆 (类似"闪回"):

```bash
python3 scripts/memory_core.py recall "关键词" --min-weight 0.7
```

### 4. 渐进式回忆

在实际对话中，**不要一次返回所有结果**:

```
用户: 我之前跟你讨论过Python的事吧？

AI (第一轮):
  → 检索 recall "Python讨论" --limit 3
  → 返回最高分的 1 条
  → "对！我记得你之前提到过[记忆内容]..."

用户: 还有呢？

AI (第二轮):
  → 返回接下来的 2 条
  → "还有一点，你还说过..."

用户: 嗯嗯继续...

AI (第三轮):
  → recall "Python讨论" --expand
  → 联想扩散，返回关联记忆
  → "啊对了！说到Python，我突然想到你还提过..."
```

## 遗忘行为模拟

当记忆的:
- **近因分 < 0.2** (超过3个月未检索): 返回时添加不确定性
- **recall_count = 0** 且创建时间很久远: "我隐约记得……"
- **综合分 < 0.5**: 不提为一条确定的记忆

示例:

> "我不太确定，但隐约记得你好像说过……"
> "我模糊地记得你很久前提到过一个相关的……"

## 检索建议

| 场景 | 参数 |
|------|------|
| 用户提到模糊概念 | `recall "概念" --limit 3` |
| 用户追问详情 | `recall "概念" --limit 5 --expand` |
| 需要精确回忆 | `recall "精确词" --min-weight 0.6` |
| 话题转换时主动闪回 | `recall "新话题" --limit 2 --expand` |
