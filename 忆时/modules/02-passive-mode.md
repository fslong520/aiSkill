# 模块 02 - 被动模式（对话中触发检索）

## 触发场景

当用户在对话中出现以下行为时触发记忆检索:

- **显式询问**: "我说过…"、"我记得之前…"、"你记得吗…"
- **话题回溯**: 回到之前的话题、延续之前的讨论
- **关键词命中**: 对话中出现已存储记忆中的关键字
- **情绪波动**: 用户表达强烈情绪，触发情绪锚定检索
- **上下文缺失**: 回答需要之前的背景信息

## 执行流程

### 1. 判断是否需要检索

分析当前对话内容，构建检索查询:

- 如果用户直接要求回忆: 使用用户原话作为查询
- 如果检测到关联话题: **提取2-3个核心词**作为查询
- 如果情绪发生变化: 以当前情绪关键词 + 情绪类型检索

### 2. 执行检索

```bash
# 基础检索 - 返回最相关的N条
python3 scripts/memory_core.py recall "查询内容" --limit 5

# 带联想扩散的深度检索
python3 scripts/memory_core.py recall "查询内容" --limit 5 --expand

# 按类型过滤
python3 scripts/memory_core.py recall "偏好" --type-filter preference

# 只检索高情绪记忆
python3 scripts/memory_core.py recall "情感" --min-weight 0.7
```

### 3. 渐进式回忆策略

**不要一次性倒出所有结果！**

1. **第一轮**: 只返回得分最高的 1-2 条
   - 自然地在回复中提及: "你之前提到过…"
2. **用户追问**: 返回接下来的 2-3 条
   - "还有这些相关的…"
3. **持续追问**: 再返回剩余结果

### 4. 遗忘曲线行为

当记忆的 recall_count 很低且创建时间很久远时:

- 返回时表达**不确定性**: "我模糊记得你之前好像提到过…"
- 提供**概要而非精确内容**: 给出关键词而非完整引用
- 如果分数低于阈值: "这方面的记忆有些模糊了…"

### 5. 记忆涌现处理

当 `recall --expand` 返回标记为 `[关联]` 的记忆时:

**主动说出**，而不是等用户问:

> "说到这个我突然想到一件事…"
> "这让我联想到你之前说过的另一个相关的事…"
> "对了，你还记得那次…"

涌现条件:
- 关联分数 > 0.6
- 与当前话题有语义关联
- 不是已经在此对话中提及过的内容
