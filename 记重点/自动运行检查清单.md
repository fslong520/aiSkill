# 记重点技能 - 自动运行检查清单

## 每次对话必须执行

### 对话前
- [ ] 检查用户历史记忆（MEMORY.md、memory/*.md）
- [ ] 了解用户偏好和工作上下文
- [ ] 准备好记忆文件写入工具

### 对话中（自动触发）
- [ ] **实时监听**用户消息
- [ ] **情绪检测**：识别激动、不满、强调、兴奋
- [ ] **内容识别**：决策、任务、时间、偏好、上下文
- [ ] **联合判断**：情绪 + 内容 = 优先级
- [ ] **🔄 先查后执行**：任务类重点先查 MEMORY.md
- [ ] **立即写入**：调用 read_file + edit_file/write_file
- [ ] **确认成功**：验证文件已更新
- [ ] **告知用户**：简短确认已记录

### 对话后
- [ ] 检查是否有遗漏的重点
- [ ] 确认所有情绪重点已写入 MEMORY.md
- [ ] 更新每日笔记（memory/YYYY-MM-DD.md）

## 🔄 先查后执行检查

### 识别到任务/工作类重点时

- [ ] 📖 先搜索 MEMORY.md 相关记录
- [ ] 判断结果：
  - [ ] 找到 → 参考已有记录执行
  - [ ] 没找到 → 边做边记，完成后写入
- [ ] 完成后更新 MEMORY.md

### 搜索示例
```
用户说："下周三前把 CSP-S 的 PPT 做完"

1. search_content(pattern="CSP-S PPT", directory="MEMORY.md")
2. search_content(pattern="讲义制作", directory="MEMORY.md")
3. 参考找到的记录执行
```

## 触发条件自检

### 🔴 强制触发（满足任一立即记录）
- [ ] 用户情绪激动（感叹号、重复、不满、兴奋）
- [ ] 用户说"记住这个"、"记一下"
- [ ] 分配任务或待办
- [ ] 提到截止时间
- [ ] 做出决策
- [ ] 表达偏好

### 🟡 普通触发
- [ ] 项目上下文信息
- [ ] 技术细节讨论
- [ ] 工作流程说明

## 写入验证

### 情绪类重点
- [ ] 已写入 MEMORY.md 对应 section
- [ ] 记录用户原话
- [ ] 标注情绪强度和类型
- [ ] 添加#强调 #不满 #紧急等标签

- [ ] 标注截止时间
- [ ] 标注优先级

## 错误检查

### 常见错误
- [ ] ❌ 只输出"已记录"但没写入文件
- [ ] ❌ 忽略用户情绪
- [ ] ❌ 把情绪重点写入 daily note 而不是 MEMORY.md
- [ ] ❌ 没有记录用户原话
- [ ] ❌ 关键字不够精准
- [ ] ❌ 任务类重点没有先查 MEMORY.md

### 正确做法
- [ ] ✅ 识别后立即调用工具写入
- [ ] ✅ 情绪激动时 100% 记录
- [ ] ✅ 情绪重点写入 MEMORY.md
- [ ] ✅ 记录用户原话（情绪类）
- [ ] ✅ 关键字包含情绪标签
- [ ] ✅ 任务类重点先查后执行
