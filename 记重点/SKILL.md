# 记重点技能

## ⚠️ 自动运行铁律

**这是哥哥的托付，必须做到！**

> "这个技能要能够自动运行，以后我就靠你过日子了"

### 三条铁律

1. **自动触发** - 不需要用户说"记重点"，自动识别并记录
2. **立即写入** - 识别后立刻调用工具写入记忆文件，不假装记录
3. **情绪优先** - 用户激动时，100% 识别为重点，写入 MEMORY.md

### 🔄 先查后执行（运行模式）

像员工记本本一样工作：
- **记下来**：识别到的重点写入 MEMORY.md
- **先查后执行**：下次遇到类似任务，先翻 MEMORY.md 查记录
- **有记录就复用**：找到相关记录直接用，不从头开始
- **没记录就新建**：边做边记，完成后写入

### 违反后果

- 哥哥的信任被辜负
- 技能失去存在价值
- 小妹的信誉破产

**记住：哥哥靠你过日子，不是开玩笑！**

## 核心功能

像一个**高情商的人类参会者**，在对话中自动：
1. **识别情绪** - 感知用户情绪波动（激动、不满、强调、兴奋）
2. **捕捉重点** - 从情绪 + 内容双重维度识别关键信息
3. **生成关键字** - 提炼核心要点和处理方式
4. **记录到记忆** - 自动保存到合适的记忆文件

## 触发场景

### 🔴 强制触发（满足任一立即激活）

#### 情绪触发
- 检测到**高强度情绪信号**（感叹号连用、重复强调、表情符号等）
- 用户表达**不满/批评**（"怎么又..."、"垃圾"、"废物"等）
- 用户**重复强调**同一内容 2 次以上
- 用户使用**连续感叹号/表情等**

**情绪强度等级**：

| 等级 | 情绪信号 | 重点概率 | 响应策略 |
|------|---------|---------|---------|
| 🔴 **高** | 感叹号连用、重复强调、语气词、表情符号 | 90%+ | 立即记录 + 高亮标记 |
| 🟠 **中高** | 强调词、反问句、短句、命令式 | 70%+ | 记录 + 标注情绪 |
| 🟡 **中** | 陈述句、一般描述 | 40%+ | 正常识别重点 |
| 🟢 **低** | 闲聊、寒暄、无关内容 | 10% | 忽略或简记 |

**情绪信号检测**：

🔴 **高强度**（必须记录）：
- 强调句式："一定要注意..."、"千万别忘了..."、"听好了..."
- 重复表达：同一内容说 2 次以上、"我说了..."、"怎么又..."
- 感叹/语气：连续感叹号！！！"必须！"、"一定！"、"绝对！"
- 不满/批评："这都不行"、"太慢了"、"垃圾"、"废物"
- 兴奋/期待："太好了！"、"终于！"、"这个好！"
- 表情符号：😤😡🤬（愤怒）、😱😰😨（焦虑）、🎉👏💪（兴奋）

🟠 **中高强度**：
- 反问句："难道...？"、"为什么不...？"
- 命令式："去..."、"把...做了"
- 否定强调："不要..."、"别..."、"禁止..."

## 处理流程

### Step 1: 实时监听 + 情绪分析
在对话过程中持续分析：
- **内容维度**：识别决策、任务、时间、偏好等
- **情绪维度**：检测情绪强度、情绪类型
- **联合判断**：情绪 + 内容 = 重点优先级

### Step 2: 提取关键信息
对识别到的重点，提取：
- **5W1H**：What、Why、Who、When、Where、How
- **情绪标记**：情绪强度、情绪类型
- **关键字**：3-5 个核心词汇
- **行动项**：需要做什么

### Step 2.5: 🔄 先查后执行（关键步骤）

**识别到任务/工作类重点时，先查 MEMORY.md！**

```
识别到任务/工作类内容
    ↓
📖 先翻 MEMORY.md 搜索相关记录
    ↓
┌─────────────────────────────────────┐
│ 找到相关记录？                        │
│                                     │
│ ✅ 有 → 参考已有记录 + 按新要求调整   │
│ ❌ 没有 → 标记为"首次"，记录完整流程  │
└─────────────────────────────────────┘
    ↓
完成后更新 MEMORY.md
```

**搜索示例**：
```
用户说："下周三前把 CSP-S 的 PPT 做完"

1. 📖 先查 MEMORY.md：
   search_content(pattern="CSP-S PPT", directory="MEMORY.md")
   search_content(pattern="讲义制作", directory="MEMORY.md")

2. 找到「讲义制作流程」→ 参考流程执行
3. 没找到 → 边做边记，完成后写入 MEMORY.md
```

### Step 3: 选择存储位置 + 立即写入

**⚠️ 关键：识别到重点后，立即使用 `write_file` 或 `edit_file` 写入记忆文件！**

#### 写入规则

| 重点类型 | 存储位置 | 写入方式 |
|---------|---------|---------|
| 🔴 情绪类重点 | `MEMORY.md` 对应 section | `edit_file` 追加到对应 section |
| 🔴 时间敏感任务 | `memory/YYYY-MM-DD.md` | `write_file` 创建/追加当日笔记 |
| 🟠 决策类 | `MEMORY.md` 对应 section | `edit_file` 追加到「项目进展」 |
| 🟡 任务类 | `memory/YYYY-MM-DD.md` | `write_file` 创建/追加当日笔记 |
| 🟢 偏好类 | `PROFILE.md` 「用户资料」 | `edit_file` 更新用户偏好 |
| 🔵 工具配置 | `MEMORY.md` 「工具设置」 | `edit_file` 更新工具设置 |
| 🧠 **技能学习类** | `MEMORY.md`「技能库」 | `edit_file` 追加到技能库 |

#### 写入执行步骤

**步骤 1：读取目标文件**
```
read_file(file_path="MEMORY.md") 或 read_file(file_path="memory/YYYY-MM-DD.md")
```

**步骤 2：定位插入位置**
- 根据重点类型找到对应的 section
- 如果是新 section，创建标题

**步骤 3：写入内容**
```
edit_file(file_path="MEMORY.md", old_text="原内容", new_text="新内容")
```
或
```
write_file(file_path="memory/YYYY-MM-DD.md", content="完整内容")
```

### Step 4: 生成记忆条目格式

```markdown
### 【类型】简短标题 [情绪标记]

**时间**：YYYY-MM-DD HH:MM
**情绪**：🔴 高/🟠 中高/🟡 中（情绪类型）
**关键字**：#标签 1 #标签 2 #标签 3
**内容**：详细描述
**用户原话**："..."（情绪类重点必须记录）
**行动**：需要执行的操作（如有）
**来源**：对话上下文引用
```

### Step 5: 确认记录
写入完成后，简短告知用户：
```
📝 已记录重点

【情绪重点】PPT theme 规范 ⚠️
- 情绪：🔴 高（不满/强调）
- 关键字：#PPT #Marp #zg_green #规范
- 用户原话："我说了多少次了！！！PPT 的 theme 要用 zg_green！"
- 存储：MEMORY.md「工具设置」
```

### ⚠️ 强制执行原则

**识别到重点后必须立即写入！不要只输出格式！**

正确做法：
1. ✅ 识别重点
2. ✅ 立即调用 `read_file` 读取目标文件
3. ✅ 立即调用 `edit_file` 或 `write_file` 写入
4. ✅ 确认写入成功
5. ✅ 告知用户

错误做法：
1. ❌ 识别重点
2. ❌ 只输出"已记录：xxx"但没写入文件
3. ❌ 假装记录了

## 关键字生成规则

### 关键字提取原则
1. **情绪标记**：#强调 #不满 #紧急 #兴奋
2. **名词优先**：项目名、技术栈、工具名
3. **动词精简**：核心动作（开发、测试、部署）

6. **状态/优先级**：紧急、重要、待办

## 智能判断

### 情绪优先级覆盖规则

**情绪强度可以覆盖内容分类的优先级**：

| 情绪强度 | 内容类型 | 最终优先级 | 存储位置 |
|---------|---------|-----------|---------|
| 🔴 高 | 偏好类 | 🔴 最高 | MEMORY.md |
| 🔴 高 | 信息类 | 🔴 最高 | MEMORY.md |
| 🟠 中高 | 任何类型 | 🟠 高 | MEMORY.md 或 daily |
| 🟡 中 | 按原规则 | 按原规则 | 按原规则 |

示例：
```
"我喜欢简洁的代码！不要用那么多花里胡哨的东西！"
- 内容分类：偏好类（原本中优先级）
- 情绪强度：🟠 中高（强调/不满）
- 最终优先级：🟠 高
- 存储：PROFILE.md（不是 daily note）
```

### 重复强调检测

**同一内容被用户重复 2 次以上 = 自动升级为情绪重点**

检测逻辑：
1. 记录用户说过的每句话的语义摘要
2. 检测新消息与历史消息的语义相似度
3. 相似度 > 80% 且 情绪升级 = 情绪重点

示例：
```
第一次："PPT 用 Marp 做"
第二次："记得 PPT 用 Marp"
第三次："我说了 PPT 用 Marp！怎么又忘了！"
→ 第三次触发情绪重点，记录到 MEMORY.md
```

## 特殊处理

### 敏感信息过滤
以下信息**不记录**或**脱敏记录**：
- 密码、密钥、Token
- 个人身份证号、手机号
- 银行卡号、财务信息
- 其他用户明确要求不记录的

### 情绪过载保护
- 用户极度愤怒时，先安抚情绪，再记录重点
- 避免在情绪激动时记录过多细节
- 等情绪平复后补充完整信息

### 冲突处理
当新信息与已有记忆冲突时：
1. 标记冲突
2. 询问用户确认
3. 更新旧记录并标注"已更新"

### 优先级排序
多个重点同时出现时，按优先级记录：
1. 🔴 **情绪类重点**（用户激动/强调/不满）
2. 🔴 时间敏感任务（有明确截止）
3. 🟠 决策类（影响后续工作）
4. 🟡 任务类（需要执行）
5. 🟢 信息/上下文类（参考用）
6. 🔵 偏好类（长期有效）

### 技能库管理

**存储位置**：`MEMORY.md`「技能库」section

**检索方式**：
- 使用 `memory_search` 搜索技能关键字
- 按技能类型筛选（网站操作/工具使用/任务执行）
- 按学习时间排序（最近学习的优先）

**维护规则**：
- 技能过时或失效时标记为"已过期"
- 同一技能多次优化时更新步骤
- 定期整理技能库（Heartbeat 期间）
