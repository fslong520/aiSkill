# 记重点技能

## 核心功能

像一个高效的人类参会者，在对话中自动：
1. **识别重点** - 捕捉关键信息、决策、任务、承诺
2. **生成关键字** - 提炼核心要点和处理方式
3. **记录到记忆** - 自动保存到 `memory/YYYY-MM-DD.md` 或 `MEMORY.md`

## 触发场景

### 自动触发（满足任一即激活）
- 对话中出现**决策性内容**（"我们决定..."、"就这样定了..."）
- 分配**任务或待办**（"你负责..."、"记得..."、"下次..."）
- 提到**时间敏感信息**（"下周三前"、"明天"、"下周"）
- 用户表达**偏好或需求**（"我喜欢..."、"我需要..."、"不要..."）
- 讨论**项目上下文**（技术细节、工作流程、业务逻辑）
- 出现**数字、日期、人名、地点**等实体信息
- 用户说"记住这个"、"记一下"、"重点是..."

### 手动触发
- 用户明确指令："记重点"、"总结一下"、"把刚才的记下来"

## 重点识别分类

### 📌 决策类（高优先级）
**信号词**：决定、确定、就这样、定了、同意、批准、选择

**示例**：
- "我们决定用 Django 而不是 Flask"
- "就这样定了，下周一开始做"
- "同意这个方案"

**记录格式**：
```
【决策】[日期] [决策内容]
- 上下文：...
- 影响范围：...
- 后续行动：...
```

### ✅ 任务类（高优先级）
**信号词**：你负责、记得、下次、待办、要、需要、完成

**示例**：
- "你负责把 PPT 做完"
- "记得下周三前提交"
- "需要更新文档"

**记录格式**：
```
【任务】[日期] [任务内容]
- 负责人：...
- 截止时间：...
- 优先级：...
- 状态：待办/进行中/已完成
```

### 💡 信息类（中优先级）
**信号词**：注意、重要的是、关键是、记住、别忘

**示例**：
- "重要的是要先测试"
- "关键是性能优化"
- "注意这个 API 的限制"

**记录格式**：
```
【信息】[日期] [信息主题]
- 关键点：...
- 应用场景：...
- 注意事项：...
```

### 👤 偏好类（中优先级）
**信号词**：我喜欢、我不喜欢、我希望、不要、最好

**示例**：
- "我喜欢简洁的代码风格"
- "不要用 rm，用 trash"
- "最好用中文注释"

**记录格式**：
```
【偏好】[日期] [偏好主题]
- 内容：...
- 适用范围：...
```

### 📋 上下文类（中优先级）
**信号词**：项目是、我们是、这个系统、工作流程

**示例**：
- "项目是 Django 博客系统"
- "工作流程是先 review 再合并"
- "这个系统处理订单"

**记录格式**：
```
【上下文】[日期] [上下文主题]
- 描述：...
- 相关文件：...
- 关联信息：...
```

### ⏰ 时间类（高优先级）
**信号词**：明天、下周、几点、之前、之后、截止

**示例**：
- "下周三前完成"
- "明天上午 9 点开会"
- "3 月 15 日之前提交"

**记录格式**：
```
【时间】[日期] [时间敏感事项]
- 时间点：...
- 事项：...
- 提醒：...
```

## 处理流程

### Step 1: 实时监听
在对话过程中持续分析用户消息，识别重点信号。

### Step 2: 提取关键信息
对识别到的重点，提取：
- **5W1H**：What、Why、Who、When、Where、How
- **关键字**：3-5 个核心词汇
- **行动项**：需要做什么

### Step 3: 生成记忆条目
按以下格式生成记忆内容：

```markdown
### 【类型】简短标题

**时间**：YYYY-MM-DD HH:MM
**关键字**：#标签 1 #标签 2 #标签 3
**内容**：详细描述
**行动**：需要执行的操作（如有）
**来源**：对话上下文引用
```

### Step 4: 选择存储位置
- **临时/日常信息** → `memory/YYYY-MM-DD.md`
- **长期重要信息** → `MEMORY.md` 对应 section
- **用户偏好** → `PROFILE.md` 「用户资料」section
- **工具配置** → `MEMORY.md` 「工具设置」section

### Step 5: 确认记录
记录完成后，简短告知用户：
```
📝 已记录：[重点标题]
   关键字：#标签 1 #标签 2
   存储：memory/2026-03-13.md
```

## 关键字生成规则

### 关键字提取原则
1. **名词优先**：项目名、技术栈、工具名
2. **动词精简**：核心动作（开发、测试、部署）
3. **时间标记**：日期、截止、周期
4. **人物/角色**：负责人、相关方
5. **状态/优先级**：紧急、重要、待办

### 示例
| 原句 | 关键字 |
|------|--------|
| "下周三前把 PPT 做完，用 Marp 制作" | #PPT #Marp #下周三 #待办 |
| "项目是 Django 博客，部署在 Ubuntu 服务器" | #Django #博客 #Ubuntu #部署 |
| "我喜欢简洁的代码，不要太多注释" | #代码风格 #简洁 #偏好 |

## 智能判断

### 什么时候记到 daily note vs 长期记忆

**记到 `memory/YYYY-MM-DD.md`**：
- 临时讨论
- 一次性任务
- 当天有效信息
- 待确认事项

**记到 `MEMORY.md`**：
- 长期有效的决策
- 工作流程/规范
- 技术选型
- 持续性的偏好

**记到 `PROFILE.md`**：
- 用户个人信息
- 工作偏好
- 沟通风格
- 称呼/代词

## 特殊处理

### 敏感信息过滤
以下信息**不记录**或**脱敏记录**：
- 密码、密钥、Token
- 个人身份证号、手机号
- 银行卡号、财务信息
- 其他用户明确要求不记录的

### 冲突处理
当新信息与已有记忆冲突时：
1. 标记冲突
2. 询问用户确认
3. 更新旧记录并标注"已更新"

### 优先级排序
多个重点同时出现时，按优先级记录：
1. 🔴 时间敏感任务（有明确截止）
2. 🟠 决策类（影响后续工作）
3. 🟡 任务类（需要执行）
4. 🟢 信息/上下文类（参考用）
5. 🔵 偏好类（长期有效）

## 记忆维护

### 每日回顾（Heartbeat 触发）
- 浏览当天 `memory/YYYY-MM-DD.md`
- 识别需要转移到 `MEMORY.md` 的长期信息
- 标记已完成的任务
- 清理过时信息

### 每周整理
- 汇总本周所有重点
- 更新项目进展
- 归档已完成事项
- 生成周报摘要

## 搭配使用

- `memory_search` - 检索已记录的重点
- `cron` - 为时间敏感任务设置提醒
- `HEARTBEAT.md` - 定期回顾重点事项
