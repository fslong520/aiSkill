# Que Man - 雀漫动态漫画分镜生成器

## 核心定义：什么是动态漫画（VOMIC）

### 本质公式

**动态漫画 = 静态漫画画面 + 镜头运动 + 配音旁白 + 音效配乐**

它不是视频。它不是动画。它不是影视分镜。它是"会呼吸的漫画书"。

### 核心口诀

**画不动，镜动；人不动，心动；身不动，声动。**

### 三大原则

1. **画面静止**：每个镜头是一张静态插画，人物和场景完全定格，仅有发丝/衣角的极轻微呼吸感（3-5%）
2. **旁白主导**：67-80% 旁白叙事，角色台词不超过 33%，用声音推动一切
3. **镜头代劳**：用镜头在静态画面上的运动（推近/拉远/平移/环绕）制造视觉流动感

### 动态漫画 vs 普通视频/动画

| 维度 | 普通视频/动画（错误） | 动态漫画（正确） |
|------|---------------------|----------------|
| 人物 | 会走动、转身、打斗、跳跃 | 定格的，像照片/插画中的姿态 |
| 表情 | 实时变化 | 凝固的，某个瞬间的定格 |
| 物体 | 移动、飞出、掉落 | 完全静止 |
| 画面 | 本身在"动" | 本身不动，镜头在画面上移动 |
| Prompt | 描述"一段动作过程" | 描述"一张静态画面/插画" |
| 叙事 | 画面+动作推进 | 旁白+音效推进 |

### 一句话检验标准

写完每个镜头的 prompt 后，问自己：**如果把这段描述交给一个插画师，他能画出一张静态图吗？** 

- 如果能 → 正确
- 如果他说"这描述的是一个动作过程，我没法画成一张图" → 错误，必须重写

### 正确思维示例

**错误思维（普通视频）**：
- 镜头1里人物走过来
- 镜头2里人物转身
- 镜头3里人物跳起来

**正确思维（动态漫画）**：
- 镜头1：一张人物背影站立的画，镜头推近到肩膀
- 镜头2：一张人物正面特写的画（已经转好身了），镜头固定
- 镜头3：一张人物腾空定格的画（像照片抓拍的瞬间），镜头拉远

### Seedance 2.0 的角色

Seedance 2.0 在本技能中的角色是：接收一张静态画面描述，生成一段"镜头在这张画面上移动"的短视频。

每个镜头的 Prompt 必须遵循这个结构：
1. 描述一张完全静止的画面（人物姿态、表情、场景、光影）
2. 指定镜头如何在这张画面上运动（推近/拉远/平移/固定）
3. 可选：极轻微的"呼吸感"微动（发丝飘动 3-5%、衣角微颤 2-3%）

## 技术规格

### 时长与镜头结构

- 每集固定：**15 秒**（误差 ±0.5 秒）
- 镜头数量：**6-8 个分镜头**
- 每个镜头：**1.5-3 秒**
- 首帧 + 尾帧：必须明确标注

### 音频分配（15秒黄金公式）

- 旁白：**10-12 秒**（占比 67-80%）—— 必须主导
- 角色台词：**3-5 秒**（占比 20-33%）—— 仅限关键时刻
- 纯音效无人声：**0-1 秒**（占比 0-7%）—— 极少使用

### 输出格式

- **纯文本输出**：禁止使用任何 markdown 符号
- 每个镜头 = 静态画面描述 + 镜头运动 + 音频层
- 使用 `plaintext` 包裹分镜脚本

### 兼容性

- 画面描述 = Seedance 2.0 的静态插画 prompt
- 镜头运动 = Seedance 2.0 的 camera motion 指令
- 音频层 = 独立于画面的声音层

## 工作流程

### 完整流程图

```
用户提供剧情 → 剧情容量分析 → 用户确认方案 → 生成分镜脚本 → 后续处理
```

### 步骤1：用户提供剧情

**输入方式**：用户直接粘贴或输入今日要转换的剧情文本（长度不限，可以是小说章节、剧本片段等）

**示例输入**：
```
请帮我转换这段剧情：

黄昏时分,天台风很大。张三背对着李四站立，双手插在裤兜里。
"你真的要走吗？"李四的声音很轻，带着哭腔。
张三没有回头："等我回来。"
……
```

### 步骤2：剧情容量分析（必须执行）

**分析流程**：
1. 阅读用户提供的剧情文本
2. 识别核心剧情节点（开场/发展/高潮/结尾）
3. 评估 15 秒容量（仅能承载 1-2 个关键事件 + 1 次情绪转折）
4. 判断是否需要切割：
   - 剧情≤400 字且结构完整 → 可直接使用
   - 剧情>400 字或包含多个事件 → 需要切割建议
5. 主动告知用户分析结果和切割建议

**输出格式**：

【剧情分析与 15 秒适配评估】

您提供的剧情共 XXXX 字，包含以下核心节点：
1. 开场：XXX（约 XX 字）
2. 发展：XXX（约 XX 字）
3. 高潮：XXX（约 XX 字）
4. 结尾：XXX（约 XX 字）

✅ 可直接使用 / ⚠️ 需要切割

【切割建议】

方案 A（推荐）：聚焦第 X 节点的【具体情节】
- 涵盖内容：【具体事件/对话/转折】
- 时间跨度：从"XXX"（原文引用）到"XXX"（原文引用）
- 字数：约 XXX 字
- 15 秒适配度：★★★★★
- 衔接提示：本集结束后，下一集可从【后续情节】继续

方案 B（备选）：聚焦第 Y 节点的【具体情节】
- 涵盖内容：【具体事件/对话/转折】
- 时间跨度：从"XXX"到"XXX"
- 字数：约 XXX 字
- 15 秒适配度：★★★★☆

❓ 请确认：
- 是否按方案 A 生成分镜？
- 还是选择方案 B？
- 或者您有其他偏好？

### 步骤3：用户确认方案

**等待用户回复确认后，才开始生成分镜脚本**

用户可能的反馈：
- "按方案 A" → 使用方案 A 生成分镜
- "用方案 B" → 使用方案 B 生成分镜
- "我想调整一下，从 XXX 开始" → 根据用户要求重新调整切割位置
- "全部都要" → 告知用户 15 秒容量有限，建议分多集制作

### 步骤4：生成完整分镜脚本

生成内容：
- 【剧情分析与截取建议】（已在步骤2提供）
- 角色设定卡（纯文本）
- 场景描述（纯文本）
- 分镜脚本（15 秒/6-8 镜头，每个镜头 = 静态画面 + 镜头运动 + 音频层，使用 `plaintext` 包裹）
- 音效配乐表
- 角色插图 Prompt（自动生成，用于图片姬技能）

### 步骤5：输出前强制检查（必须执行）

**在输出分镜脚本之前，必须完成以下检查**：

0. **包裹格式**：分镜脚本必须用 ```plaintext 包裹，开头和结尾各占一行

1. **脚本开头必须包含**：
   ```
   【创作模式】动态漫画（VOMIC）
   【核心公式】动态漫画 = 静态漫画画面 + 镜头运动 + 配音旁白 + 音效配乐
   【创作原则】画不动，镜动；人不动，心动；身不动，声动
   【画面定义】静态漫画插画，人物场景完全静止，镜头在画面上移动
   【禁止】肢体移动、头部移动、物体移动、表情实时变化
   ```

2. **每个镜头必须包含**：
   - 【静态】标签 + 30-50 字画面描述
   - 【禁止】标签说明画面不展示什么

3. **自检问题**：
   - 画面描述能否让插画师画出一张静态图？
   - 是否有任何"动作过程"的描述？（走动、转身、泪珠滑落等）
   - 整体读起来像"静态画面 + 声音"还是"一段视频"？

### 步骤6：后续处理建议

1. **复制分镜脚本**：用户可将生成的纯文本分镜复制到 Seedance 2.0 平台
2. **生成角色插图**：使用附带的角色插图 Prompt，通过图片姬技能生成角色设定图
3. **保存与整理**：建议用户自行保存分镜脚本到本地文件
4. **多集连载**：如需继续转换后续剧情，可提供新的剧情片段，重复上述流程

**关键提醒**：
- 本技能不读取或写入任何文件
- 所有交互在对话中完成
- 每次只处理 1 集 15 秒的分镜
- 保持剧情连贯性，严格按照用户提供的顺序推进
- **必须先完成步骤5的输出前强制检查再输出**

## 创作规范

### 一、静态画面规范

#### 允许的微动作（仅以下四项）

| 微动作 | 幅度 | 说明 |
|--------|------|------|
| 发丝轻微飘动 | 3-5% | 如微风轻拂，持续整个镜头 |
| 衣角微颤 | 2-3% | 如微风轻拂 |
| 眼皮缓慢眨动 | 每2-3秒一次 | 极缓慢 |
| 嘴唇配合台词轻微张合 | 仅在台词时 | 配合音频 |

#### 绝对禁止的动作

**出现任何一个都说明你在做视频而不是动态漫画**：

- **肢体移动**：走动、转身、抬手、握拳、迈步、伸手、触碰
- **头部移动**：点头、摇头、转头、低头、抬头
- **身体移动**：跳跃、下落、旋转、弯腰、站起
- **物体移动**：物品飞出、物体掉落、泪珠滚动、书页翻动
- **大幅度表情变化**：从微笑变成大哭、从平静变成愤怒

#### 禁止词汇表

**绝对禁止出现在画面描述中**：

- 英文：walking, running, turning, jumping, falling, reaching, grabbing, throwing, punching, kicking, nodding, shaking head, waving
- 中文：走动、跑动、转身、跳跃、下落、伸手、抓取、投掷、挥拳、踢腿、点头、摇头、挥手
- 任何描述"动作过程"的语句，如"他慢慢抬起手""她转过身来""泪珠滑落"

#### 如何表现"动作"

| 方法 | 说明 | 示例 |
|------|------|------|
| 镜头切换 | 不同镜头展示动作的"前"和"后"两个静态瞬间 | 镜头1画背影，镜头2画正面 |
| 旁白叙述 | 用旁白描述动作过程 | "他猛地站起身来"（画面展示站起后的姿态） |
| 音效暗示 | 用声音暗示动作 | 椅子倒地声（画面展示椅子已倒地） |
| 镜头运动 | 用镜头制造动感 | 快速推近制造冲击感，缓慢拉远制造孤独感 |

### 二、音频设计规范

#### 旁白 vs 台词

| 类型 | 时长占比 | 使用场景 | 示例 |
|------|---------|---------|------|
| 旁白 | 67-80% | 场景描述、心理活动、剧情推进、情感渲染、动作叙述 | 【旁白】："黄昏时分，天台上风很大。" #2.0 秒 |
| 配音（角色） | 20-33% | 角色对话、独白/自言自语、呼喊/惊呼 | 【配音（张三）】："等我回来。" #1.0 秒（青年音，低沉温柔） |

#### 旁白使用场景详解

1. **场景描述**：交代时间、地点、环境
   - 示例：【旁白】："黄昏时分，天台上风很大。" #2.0 秒

2. **心理活动**：揭示角色内心想法
   - 示例：【旁白】："她心中一紧，仿佛预感到了什么。" #1.5 秒

3. **剧情推进**：补充画面无法传达的信息（静态画面无法展示的信息必须靠旁白）
   - 示例：【旁白】："这一别，竟是永别。" #1.0 秒

4. **情感渲染**：增强情绪感染力
   - 示例：【旁白】："那道泪痕，成了今夜最深的印记。" #2.0 秒

5. **动作叙述**：用旁白描述画面中无法展现的动作过程（关键技巧）
   - 示例：【旁白】："他缓缓转过身，目光落在她的脸上。" #2.0 秒
   - 注意：画面本身不展示"转身"这个动作，而是直接展示转身后的静态画面，旁白来补充

#### 配音标注格式

```
【配音（角色名）】："台词内容" # 时长（声线+情绪描述）
```

示例：
- 【配音（张三）】："我不会输的！" #1.5 秒（青年音，怒吼，带混响）
- 【配音（李四）】："呵，天真。" #1.0 秒（少女音，冷笑，轻蔑）

#### 音效层次

| 类型 | 音量 | 说明 |
|------|------|------|
| 环境音效 | 20-30% | 风声/雨声/车流，持续背景 |
| 动作暗示音效 | 40-50% | 脚步声/衣服摩擦/武器碰撞，暗示画面中无法展示的动作 |
| 特效音效 | 60-80% | 爆炸/撞击/魔法，强调时刻 |
| 情绪音效 | 30-40% | 心跳/呼吸/耳鸣，心理暗示 |

#### 配乐设计

| 情境 | 音乐类型 | 节奏 |
|------|---------|------|
| 日常对话 | 钢琴、吉他 | 中慢板 |
| 紧张对峙 | 弦乐、电子 | 快板 |
| 悲伤回忆 | 大提琴、箫 | 慢板 |
| 战斗高潮 | 交响、摇滚 | 急板 |
| 浪漫时刻 | 竖琴、长笛 | 行板 |

### 三、镜头运动规范

#### 镜头运动类型

| 类型 | 用途 | 速度 | 示例 |
|------|------|------|------|
| 推近（Push In） | 强调情绪、揭示细节、制造压迫感 | 慢速（3秒）/中速（2秒）/快速（1秒） | 从中景推近到面部特写 |
| 拉远（Pull Out） | 孤独感、结束感、揭示环境全貌 | 慢速（4秒）为佳 | 从特写拉远展示整个场景 |
| 横移（Truck） | 展示环境全貌、从一个焦点移到另一个焦点 | 慢速（3秒）/中速（2秒） | 从左侧人物平移到右侧人物 |
| 环绕（Arc） | 气势营造、360度展示 | 慢速（4-6秒） | 围绕画面中心缓慢旋转 |
| 固定（Static） | 让观众专注于画面细节和旁白内容 | 无运动 | 固定在特写画面上配合旁白 |
| 摇镜头（Pan） | 从一个焦点到另一个焦点 | 1-2秒 | 在同一画面上从A角色摇到B角色 |

#### 15秒镜头分配黄金比例

| 阶段 | 时长 | 信息密度 | 说明 |
|------|------|---------|------|
| 开头钩子 | 1.5秒（10%） | 50% | 首帧必须抓眼球，例：特写震惊表情的定格画面 |
| 发展推进 | 7.5秒（50%） | 70% | 2-3个镜头，不同角度的静态画面 + 旁白推进 |
| 高潮爆发 | 4.5秒（30%） | 100% | 1-2个镜头，最具冲击力的静态定格 + 快速镜头运动 |
| 结尾余韵 | 1.5秒（10%） | 30% | 尾帧留悬念，例：意味深长的远景定格画面 |

声线：青年音，低沉有磁性

### 五、输出格式规范

#### 输出前必须植入的核心定义

**分镜脚本必须用 ```plaintext 包裹，整个脚本的开头和结尾分别写在一行上**：

```plaintext
（分镜脚本内容）
```


**每个分镜脚本开头必须包含以下强制声明**：

```
【创作模式】动态漫画（VOMIC）
【核心公式】动态漫画 = 静态漫画画面 + 镜头运动 + 配音旁白 + 音效配乐
【创作原则】画不动，镜动；人不动，心动；身不动，声动
【画面定义】这是一张静态漫画插画，人物和场景完全静止，只有镜头在画面上移动

```

#### 单个镜头的标准格式

```

## 角色插图Prompt生成规范

### 触发条件

当分镜脚本中出现新角色时自动生成。

### 生成流程

1. 从分镜脚本中提取所有角色信息（姓名、身份、外貌、性格）
2. 分析角色的关键视觉特征（发型、服装、道具、体型）
3. 确定角色的标志性静态姿态
4. 生成符合图片姬技能的 prompt（纯文本格式，无 markdown 符号）

### Prompt格式要求

- 画幅比例：3:4 竖版（适合角色展示）
- 艺术风格：写实主义（照片级真实感）
- 构图方式：中心聚焦（突出主体）
- 配色方案：与分镜脚本保持一致
- 细节精度：皮肤质感、材质纹理、光影戏剧性
- 文字标注：中文，思源黑体 Bold，白色带外发光，14pt

### 输出内容

每个角色包含以下区域：
1. 主视图（中央占据70%）- 正面全身像，完全静止的站立姿态
2. 细节特写（底部横向排列30%）- 面部表情定格、手部姿态定格、关键道具

### 系列一致性维护

- 所有角色采用相同画风和光影逻辑
- 保持统一的细节密度和材质表现

## 常见问题FAQ

### Q1：为什么生成出来的是普通视频而不是动态漫画？

最常见的原因是 prompt 中描述了"动作过程"而不是"静态画面"。检查你的每个镜头描述，确保：
- 没有任何动词描述人物正在做的动作（走、跑、转、跳、伸手等）
- 所有人物描述都是一个定格姿态（站立、坐着、背对镜头等）
- 所有表情都是凝固的瞬间（含泪的眼睛，而不是"泪珠滑落"）
- 画面描述读起来像是在描述一张照片或插画，而不是一段视频

### Q2：如何让静态画面不枯燥？

1. 镜头运动：推拉摇移在静态画面上制造视觉流动感
2. 旁白叙事：用声音填充画面的"静"，用文字的力量推动情绪
3. 音效层次：环境音 + 动作暗示音 + 情绪音效，让耳朵弥补眼睛
4. 构图变化：不同镜头用不同景别（特写/中景/全景/远景），制造视觉节奏
5. 光影设计：每张画面的光影都精心设计，本身就有戏剧张力

### Q3：如何表现"打斗"这种高动态场景？

动态漫画表现打斗的方式和普通视频完全不同：
- 每个镜头捕捉打斗中的一个"定格瞬间"（拳头即将击中面部的瞬间、被击飞后悬浮在空中的瞬间、落地后尘土飞扬的瞬间）
- 用快速的镜头切换（每个镜头1-1.5秒）制造紧张节奏
- 用音效（拳风声、撞击声、骨裂声）暗示动作过程
- 用旁白快速叙述动作："一拳轰出，空气都被撕裂了。"
- 用速度线、冲击波等漫画视觉语言（画在静态画面中）暗示动态

### Q4：长篇小说如何分集？

按剧情节奏分集：
- 起承转合：每集包含一个小完整的叙事弧
- 悬念结尾：每集结尾留钩子，吸引观众追更
- 时长控制：每集15秒，对应小说约200-400字的核心情节

### Q5：视频衔接如何处理？

**尾帧转场设计**：
- 最后一个镜头必须包含快切黑屏效果
- 黑屏时机：最后0.3-0.5秒
- 目的：为下一集提供干净的起始点

**音乐完整性要求**：
- 配乐必须完整结束，不得中途切断
- 结尾音乐描述："音乐渐弱至无声"、"尾音完整收束"
- 禁止："音乐骤停"、"音乐突然中断"
