# 示例

## 示例 1：ABC457 题解

### 输入

```
用户：使用urlgo技能配合搬题姬技能搬运https://atcoder.jp/contests/abc457/tasks
      → 生成题面汇总 + 完整题包（A~E）

用户：生成abcde的题解，要包含题面，而且是启发式的题解
      → 使用析题技能生成启发式题解
```

### 产出

1. `abc457.md` —— 7 题题面汇总（搬题姬产出）
2. `abc457a_数组.zip` ~ `abc457e_交叉桌布.zip` —— 5 份完整题包（搬题姬产出）
3. `abc457_题解.md` —— 539 行启发式题解（析题产出）

### 题解特点

- 每道题从「读完题脑中一片空白」开始推演
- 展示暴力 → 撞墙 → 转向 → 优化的完整思维路径
- 代码逐行精讲（结合用户指定的标程文件）
- 每道题附常见错误分析表
- 末尾五题串讲总结

## 示例 2：ABC460 题解（今日实战）

### 用户过程

```
用户：搬运 https://atcoder.jp/contests/abc460/tasks
用户：审计标程 → 发现 D 题 BFS 对所有黑格失效 → 双 BFS 修复
用户：发现 E 题漏了 19 位数字 → k=19 修复
用户：要求题解格式参考 459 --> 重写
用户：看不懂公式 → 补数值例子
用户：要求优化析题技能 → 本文件即产出
```

### 实战中暴露的盲区（已反哺技能）

| 盲区 | 触发场景 | 加固措施 |
|------|---------|---------|
| 公式无数值例 | 用户看不懂 `≡ (mod M)` | SKILL.md + 02-write.md 新增「公式配数值例」规则 |
| 边界值遗漏 | `N=10^18` 需枚举到 19 位 | 02-write.md 新增边界值排查清单 |
| 全同输入退化 | 全黑格 BFS 奇偶法失效 | 02-write.md 新增边界清单 + 对面视角技巧 |
| 歧路不充分 | 题解直接给正解，读者不知道为什么绕路 | SKILL.md 新增歧路展示心法 |

### 产出

1. `abc460.md` —— 7 题题面汇总
2. `abc460{a..g}_*.zip` —— 7 份完整题包
3. `abc460_solutions.md` —— 题解（含公式数值例 + 歧路展示 + 双 BFS 对面视角）

### 协作模式

```
搬题姬（获取/翻译题面）→ 析题（启发式题解撰写）
     ↓                          ↓
  URL → 题面                题面 → 详解
     题面汇总               + 标程精讲
     + 完整题包              + 错误分析
                             + 串讲总结
                             + 边界排查
```
