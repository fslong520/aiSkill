# 使用示例

## 示例一：诊断

**输入**：体检技能诊所

**执行**：
```
1. 确定意图 → 诊断
2. 读取文件 → SKILL.md + modules/
3. 检查结构 → Keywords(10) + Summary(10) + Strategy(10) + AVOID(5) = 35
4. 检查触发 → metadata.trigger存在 = 10
5. 检查内容 → Token效率(13) + 信号密度(10) + 可执行性(10) = 33
6. 检查实践 → 渐进式披露(8) + Human-in-the-Loop(7) + CLI友好(5) = 20
7. 总分 → 35+10+33+20 = 98
8. 等级 → S
```

**输出**：
```
总分：98/100
等级：S
建议：保持现状
```

## 示例二：创建简单技能

**输入**：创建代码审查技能

**执行**：
```
1. 收集需求：
   - 名称：代码审查
   - 描述：审查代码质量
   - 触发：代码审查、code review
   - 输入：代码路径
   - 输出：审查报告
2. 判断 → ✅ 值得沉淀
3. 选择 → 简单模板
4. 创建 → 代码审查/SKILL.md
5. 验证 → 通过
```

**输出**：
```
已创建：代码审查/SKILL.md
```

## 示例三：创建复杂技能

**输入**：创建文档生成技能，支持多格式

**执行**：
```
1. 收集需求：
   - 名称：DocGen
   - 描述：文档自动生成
   - 触发：生成文档、docgen
   - 输入：项目路径
   - 输出：文档文件
2. 判断 → ✅ 值得沉淀
3. 选择 → 复杂模板（模块化）
4. 创建：
   - DocGen/SKILL.md
   - DocGen/modules/01-analyze.md
   - DocGen/modules/02-generate.md
   - DocGen/modules/03-export.md
5. 验证 → 通过
```

**输出**：
```
已创建：DocGen/
├── SKILL.md
└── modules/
    ├── 01-analyze.md
    ├── 02-generate.md
    └── 03-export.md
```
