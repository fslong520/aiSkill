# Phase 3：概念串联（构建知识图谱）

## 输入

- 所有章节的 chunks 列表
- 用户指定的关注方向（可选）

## 流程

### 步骤 1：跨章节概念连线

扫描所有 chunks，发现同主题 / 同概念在不同章节的关联。

**关联类型**：

| 类型 | 含义 | 示例 |
|------|------|------|
| `延续` | 前一章的概念在后一章深化 | 回溯法 → BFS |
| `对比` | 两种概念的对立或取舍 | DFS vs BFS |
| `示例` | 抽象概念的具体应用 | 递归 → 冰激凌排队 |
| `因果` | 一个概念导致另一个 | 指数爆炸 → 需要剪枝 |
| `反题` | 后一章推翻前一章的假设 | "规则管用" → "规则会失效" |

### 步骤 2：聚类为"蛇"

将相关 chunks 聚合成贯穿多章的线索。每条蛇是一条路径：

```
蛇名：从直觉到系统
ch0（直觉规则会失败）→ ch1（排除法）→ ch2（回溯法）→ ch4（极小化极大）→ ch6（剪枝）
```

**至少找到 3 条蛇**。

### 步骤 3：生成图谱

输出形式为 Markdown 概念连线表 + 蛇列表。

## 输出

- 概念连线表（最少 8 条连线）
- 蛇列表（最少 3 条）

## Human-in-the-Loop

图谱生成后，向用户展示：
> "已生成以下概念连线与线索。是否有需要增删改的连线？关注方向是否需要调整？"

用户确认后再进入下一阶段。

## AVOID

- AVOID 图谱过于稀疏（<8 条连线）
- AVOID 孤立节点——每个 chunk 至少有一条连线
- AVOID 蛇的数量过少（<3）
