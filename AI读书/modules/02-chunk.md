# Phase 2：逐章精读（提取 chunks）

## 输入

- Phase 1 输出的章节 JSON（含 chapters 数组，每章有 id / title / text）
- 用户指定的阅读目的 / 关注方向

## 流程

对每一章，**依次**执行，不可跳章：

### 步骤 1：阅读文本

读取本章 `text`，理解内容。

### 步骤 2：提取知识单元

每个 chunk 是一个独立的知识点，包含：

| 字段 | 说明 |
|------|------|
| id | `ch1_01`、`ch1_02`…… |
| title | 知识单元标题（一短句） |
| content | 要点（2-5 句） |
| tags | `核心概念` / `反直觉` / `教学技法` / `案例` / `代码` |

每章提取 **3-8 个 chunk**。

### 步骤 3：记录反直觉点

标记本章中令人意外、反常识或存疑的内容。这些往往是最有价值的知识。

## 输出

每章产出 chunks 列表，暂存。

## Human-in-the-Loop

**每章处理完后**，向用户展示该章的 chunks，询问：
> "第 X 章的知识单元是否准确？是否有遗漏或需调整之处？"

用户确认后再进入下一章。

## AVOID

- AVOID 一次处理多章
- AVOID chunk 过少（<3）或过多（>10）
- AVOID 忽略反直觉点——标记它们
