# Phase 5：成果输出

## 输入

- 校验通过的摘要内容
- 知识图谱
- 图书元数据

## 输出格式

最终产出一份 Markdown 文档，包含以下部分：

### 1. 图书元数据

| 项目 | 内容 |
|------|------|
| 书名 | |
| 作者 | |
| 总章节 | |
| 总字数 | |

### 2. 全书总览（300-500 字）

```
一句话概括 → 核心论点 → 全书脉络简述
```

### 3. 逐章精华摘要

每章：
- **章节标题**
- **核心论点**（一句话）
- **关键知识单元**（3-8 个 chunk，表格形式）
- **反直觉点/疑问**（如有）

### 4. 知识图谱

- **概念连线表**（Markdown 表格）

| 概念 A | 关系 | 概念 B | 所在章节 |
|--------|------|--------|----------|

- **蛇列表**：每条蛇是一条贯穿多章的线索

### 5. 阅读目的专项回答（如有）

针对用户指定的关注方向，从全书中提取相关观点汇总。

建议在此节开头注明：
> "以下内容针对【用户指定的关注方向】整理"

### 6. 延伸思考

- 书中未解决的问题
- 与已知知识的冲突
- 可进一步探索的方向

## 文件命名

`《书名》读书笔记.md`

## 输出路径

默认：图书所在目录
可选：用户指定路径

## Human-in-the-Loop

输出前询问用户：
> "笔记将保存为《书名》读书笔记.md。是否要调整输出路径或增减内容？"
