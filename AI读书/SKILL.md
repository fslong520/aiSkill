## Keywords

读书, 提炼, 拆书, 书摘, 知识图谱, 结构化笔记, EPUB 阅读, PDF 阅读, 摘要生成

## Summary

用户提供图书文件（EPUB/MD/TXT/PDF），AI 逐章提取关键知识单元，构建跨章节概念关联知识图谱，输出结构化摘要。无需额外配置 LLM 供应商。

## Strategy

1. **文本提取**：用 extract_book.py 解析图书（支持 EPUB/MD/TXT/PDF），按章节拆分
2. **逐章精读**：对每章提取关键知识单元（chunks）
3. **概念串联**：跨章节发现相关概念，构建知识图谱
4. **多角校验**：从不同视角审视摘要完整性
5. **成果输出**：章节摘要 + 知识图谱 + 全书总览

## Prerequisites

- PDF 解析依赖 `pdftotext`（poppler-utils），请确保已安装：
  ```bash
  sudo apt install poppler-utils    # Ubuntu/Debian
  brew install poppler              # macOS
  ```

## AVOID

- AVOID 跳章阅读，必须逐章处理
- AVOID 一次性读完所有章节再提取——每章读完立即提取，最后再串联
- AVOID 知识图谱过于稀疏——至少找到 3 条跨章节概念连线
- AVOID 摘要写成「本书介绍了……本书讲述了……」式的干瘪官腔
- AVOID 遗漏反直觉或有争议的观点——这些往往是最有价值的知识

## 工作流程

### Phase 0：用户输入

用户提供：
- 图书文件路径（支持 `.epub` / `.md` / `.txt` / `.pdf`）
- 可选：阅读目的 / 关注方向（如「关注系统设计权衡」「关注历史脉络」）

### Phase 1：文本提取

调取 extract_book.py 将图书解析为章节 JSON：

```
python3 modules/extract_book.py <input> --pretty
```

输出结构：
```json
{
  "title": "书名",
  "author": "作者",
  "total_chapters": 12,
  "chapters": [
    { "id": "ch_1", "title": "第一章标题", "text": "正文内容..." },
    ...
  ]
}
```

验证：`total_chapters` > 0，每章 `text` 不为空。

### Phase 2：逐章精读（提取 chunks）

对每一章，依次执行：

1. **阅读文本**：读取本章 `text`
2. **提取知识单元**（chunks）：每个 chunk 是一个独立的知识点
   - 一个概念、一个事实、一个论点
   - 每个 chunk 包含：`id`、`title`、`content`、`chapter`、`tags`
3. **记录疑问或反直觉点**：标记本章中令人意外、反常识或存疑的内容

输出每章的 chunks 列表，暂存。

**每章处理完后再进入下一章**，不可跳读。

### Phase 3：概念串联（构建知识图谱）

所有章节处理完毕后：

1. **跨章节概念连线**：扫描所有 chunks，发现同主题/同概念在不同章节的关联
   - 例如：第 2 章讲「缓存策略」，第 7 章讲「CDN 架构」→ 连线：缓存 → CDN
2. **聚类**：将相关 chunks 聚合成「蛇」（snake）——一条贯穿多章的线索
3. **生成图谱**：
   - 节点：每个 chunk
   - 边：概念关联（标注关联类型：`延续` / `对比` / `因果` / `示例` / `反题`）

输出形式：结构化列表或 Markdown 图谱描述。

### Phase 4：多角校验（答辩式审查）

模仿 SpineDigest 的多角色对抗机制：

**角色 A——摘要撰写者**：根据 chunks 和 snakes 撰写章节摘要和全书摘要

**角色 B——挑剔教授**：审阅上述摘要，从以下角度提出质疑：
- 「这个结论有文本支撑吗？在哪一章？」
- 「有没有反例或替代观点被忽略了？」
- 「这个概括是否过于简化？」
- 「阅读目的（用户指定的关注方向）是否被充分覆盖？」

**角色 C——答辩者**：回应质疑，补充遗漏，修正偏差

反复 1-2 轮，直到摘要经得起推敲。

### Phase 5：成果输出

最终产出一份结构化文档，包含以下部分：

#### 1. 图书元数据
- 书名、作者、总章节数、总字数

#### 2. 全书总览（300-500 字）
一句话概括→核心论点→全书脉络简述

#### 3. 逐章精华摘要
每章：
- **章节标题**
- **核心论点**（一句话）
- **关键知识单元**（3-8 个 chunk，每个含标题 + 要点）
- **反直觉点/疑问**（如有）

#### 4. 知识图谱
- **概念连线表**：

| 概念 A | 关系 | 概念 B | 所在章节 |
|--------|------|--------|----------|
| 缓存策略 | → 支撑 → | CDN 架构 | ch2, ch7 |
| 最终一致性 | ↔ 权衡 → | 强一致性 | ch4, ch6 |

- **蛇列表**：每条蛇是一条贯穿多章的线索，列出蛇的路径

#### 5. 阅读目的专项回答（如有）
针对用户指定的关注方向，从全书中提取相关观点汇总

#### 6. 延伸思考
- 书中未解决的问题
- 与已知知识的冲突
- 可进一步探索的方向

## 输出格式

最终输出为 Markdown 文档，可直接保存为 `.md` 文件，供后续回顾或分享。

文件命名约定：`《书名》读书笔记.md`

输出路径：默认为图书所在目录，或用户指定路径。
