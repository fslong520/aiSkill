# 破晓 - 智能资讯早报生成器

## 核心定义

**破晓**是一个每日晨间早报生成器，为投资人、从业者提供开盘前的决策支持。

### 两大核心板块

|| 板块 | 目标用户 | 核心价值 | 内容形式 |
|------|---------|---------|---------|
| 💰 早盘策略 | 投资人 | 具体可操作的投资建议 | 核心观点 + 操作建议表 + 利好/利空催化 |
| 📰 行业快讯 | 从业者 | 领域内最新动态 | 按领域分类的新闻卡片 |

### 核心原则（必须遵守）

⚠️ **数据真实性**：所有新闻链接必须是搜索结果中的真实URL，严禁编造链接  
⚠️ **链接有效性**：所有链接必须包含文章ID，能直接跳转到具体新闻页面，严禁使用媒体主页或列表页链接
⚠️ **标的动态性**：所有投资标的必须是当日搜索验证过的，严禁使用固定映射表  
⚠️ **建议可操作性**：投资建议必须具体可操作（买入/卖出/观望 + 仓位），不能只是罗列新闻  
⚠️ **风险提示醒目**：必须明确标注「本早报仅供参考，不构成投资建议」

## 整体流程概览

```
用户选择领域 → 多维度搜索 → 分析研判 → 生成HTML日报 → 展示核心看点
```

**每个步骤的输入输出**：

|| 步骤 | 输入 | 输出 |
|------|------|------|
| 1. 确认关注领域 | 用户需求 | 关注领域列表（默认：AI科技+半导体+娱乐八卦） |
| 2. 多维度搜索 | 关注领域 | 新闻数据（标题、摘要、来源、链接、标的） |
| 3. 新闻质量自检 | 新闻数据 | 时效性验证报告 |
| 4. 生成HTML | 三大板块内容 | 精美HTML早报页面 |
| 5. 展示结果 | HTML页面 | 核心看点总结 + 风险提示 |

## 步骤2：多维度搜索抓取资讯

### 搜索策略（每个领域至少执行 2-3 次搜索）

#### 1. 行业动态搜索

**目的**：获取领域内最新进展  
**关键词策略**：
- 国内行业：中文关键词 + 日期（如 "AI大模型 最新进展 2026 年 3 月"）
- 国际科技：英文关键词 + 日期（如 "AI LLM breakthrough news March 2026"）  
**时效性要求**：使用 `freshness: "day"` 确保当日新闻

#### 2. 投资标的搜索

**目的**：针对重要新闻搜索关联的 A 股/美股标的  
**关键词示例**："[新闻关键词] A 股 概念股 龙头 2026 年 3 月"  
**重要**：不要使用固定的标的映射表，必须根据当日新闻动态搜索

### 搜索质量标准

#### 数据采集要求

每个领域至少收集 **3-5 条有价值的新闻**，每条新闻必须包含：

|| 字段 | 说明 | 是否必须 |
|------|------|---------|
| 标题 | 新闻标题 | ✅ 必须 |
| 摘要 | 核心内容概括 | ✅ 必须 |
| 来源 | 发布媒体 | ✅ 必须 |
| 时间 | 发布时间 | ✅ 必须 |
| 原文链接 | 真实可访问的URL | ✅ 必须 |
| 影响方向 | 利好/利空 | 投资相关新闻必须 |
| 关联标的 | 相关股票代码 | 投资相关新闻必须 |
| 影响逻辑 | 为什么利好/利空 | 投资相关新闻必须 |

#### 信源优先级

优先选择权威信源：
- **国内**：财联社、证券时报、36氪、第一财经、华尔街见闻
- **国际**：Bloomberg、Reuters、TechCrunch

#### 链接格式强制校验（重要！）

**必须校验链接是否为具体新闻页面，严禁使用以下类型的链接：**

❌ **禁止使用的链接类型**：
- 信源主页链接：如 `https://36kr.com`、`https://36kr.com/newsflashes`、`https://www.yicai.com/news/`
- 列表页链接：以 `/news/`、`/articles/`、`/hot/`、`/list/` 结尾的URL
- 首页重定向链接：如 `example.com/` 或 `example.com`
- 占位符链接：如 `#`、`javascript:void(0)`、`example.com`

✅ **正确的新闻链接特征**：
- 包含文章ID或唯一标识符，如：
  - 36氪：`https://36kr.com/p/1234567890` 或 `https://36kr.com/news/1234567890`
  - 第一财经：`https://www.yicai.com/news/1234567.html` 或 `https://www.yicai.com/brief/1234567.html`
  - 财联社：`https://www.cls.cn/detail/1234567` 或 `https://www.cls.cn/telegraph/1234567`
  - 华尔街见闻：`https://wallstreetcn.com/articles/1234567`
  - 证券时报：`https://www.stcn.com/article/detail/1234567.html`
- URL结构中必须包含具体的数字ID或文章slug

**校验方法**：
1. 从Bing搜索结果的 `url` 字段直接获取链接（不得手动构造或修改）
2. 检查URL是否符合具体文章格式（必须包含数字ID或唯一标识符）
3. 如果链接是主页或列表页，必须从搜索结果中重新选择带有文章ID的链接
4. 对于每条新闻，必须完整记录搜索结果中返回的URL，不得截断、替换或简化

#### 质量保障

- 如果某个领域搜索结果不足，尝试换关键词重新搜索
- 确保所有链接真实可访问，严禁使用 `#` 或 `example.com`
- 每条新闻链接必须能直接跳转到该新闻的原文页面，不是媒体主页

### 并行执行要求

不同领域的搜索之间没有依赖关系，**必须并行执行**以提高效率。

## 步骤 3：新闻质量自检（必须执行！）

### ⚠️ 重要：生成前必须自检新闻时效性！

在生成 HTML 早报之前，必须对所有收集的新闻进行时效性检查，确保都是**今日最新新闻**。

### 自检标准

|| 检查项 | 标准 | 不通过处理 |
|---------|------|-----------|
| 新闻时间 | 必须是今日（当天）或昨日 | 超过 2 天的新闻 → 重新搜索 |
| 链接有效性 | 链接必须包含文章 ID | 无效链接 → 重新搜索 |
| 内容相关性 | 与所选领域强相关 | 无关内容 → 剔除 |
| 信息完整性 | 标题 + 摘要 + 来源 + 时间 + 链接 | 缺失字段 → 补充或剔除 |

### 时效性判断方法

1. **查看新闻来源显示的时间**
   - "X 小时前"、"今日"、"昨天" → ✅ 通过
   - "2 天前" → ⚠️ 谨慎使用（仅限重大事件）
   - "3 天前"及更早 → ❌ 不通过，重新搜索

2. **检查 URL 中的日期**
   - URL 中包含当日日期（如 20260311、2026/03/11） → ✅ 优先使用
   - URL 中日期超过 2 天 → ❌ 不通过

3. **搜索验证**
   - 对存疑的新闻，用 Bing 重新搜索标题
   - 如果搜索结果都是旧闻 → ❌ 不通过，更换关键词重新搜索

### 自检不通过的处理流程

```
发现老旧新闻 → 分析原因（关键词不精准/搜索范围太窄） 
→ 调整搜索关键词 → 重新搜索 → 验证新新闻时效性 
→ 通过后继续生成
```

### 自检清单（生成前逐项检查）

- [ ] 所有新闻都是今日或昨日的
- [ ] 所有链接都能直接跳转到具体文章（不是媒体主页）
- [ ] 每条新闻都有明确的来源和时间
- [ ] 投资建议有具体标的和操作建议（买入/卖出/观望 + 仓位）
- [ ] 利好/利空催化有明确的逻辑链条

**如果自检不通过，必须重新搜索，不能凑合生成！**

## 步骤4：分析研判与内容组织

基于搜索到的所有资讯，进行深度分析和内容组织。

### 一、投资分析板块（核心）

#### 1. 核心观点提炼

**要求**：用一句话概括今日最值得关注的投资方向  
**格式**：明确看多/看空/震荡 + 核心逻辑  
**示例**：「今日半导体板块看多，受国产替代加速和AI芯片需求激增双重利好驱动」

#### 2. 操作建议表

筛选 **3-5 个最值得关注的标的**，每个标的必须包含：

|| 字段 | 说明 | 格式要求 |
|------|------|---------|
| 股票代码 + 名称 | 股票标识 | A股用6位数字代码，美股用字母代码 |
| 方向 | 操作建议 | 买入 / 卖出 / 观望（三选一） |
| 建议仓位 | 仓位控制 | 具体百分比（如"10%"、"观望"） |
| 核心逻辑 | 为什么 | 一句话说清楚 |

**示例**：
```
|| 标的 | 方向 | 仓位 | 核心逻辑 |
|------|------|------|---------|
| 300661.SZ 圣邦股份 | 买入 | 15% | 国产模拟芯片龙头，受益于汽车电子需求爆发 |
| NVDA | 观望 | - | 财报前不确定性高，等待业绩指引明确 |
```

#### 3. 利好/利空催化梳理

**利好信号关键词**：突破、首发、领先、创新、融资、增长、超预期、获批、大单  
**利空信号关键词**：监管、处罚、下架、亏损、裁员、竞争、替代、预警、暴雷

**要求**：每条催化必须关联到具体标的

**示例**：
```
利好催化：
- ✅ 寒武纪MLU芯片获大单（利好：300661.SZ 圣邦股份）
- ✅ 特斯拉FSD中国落地（利好：002594.SZ 比亚迪）

利空预警：
- ⚠️ 光刻机出口限制升级（利空：688396.SH 华润微）
```

### 二、行业快讯板块

#### 组织方式

按用户选择的领域分类，每个领域选出最重要的 **2-4 条新闻**

#### 新闻卡片格式

每条新闻必须包含：

|| 字段 | 要求 | 示例 |
|------|------|------|
| 标题 | 精炼有力，15字以内 | "OpenAI发布GPT-5" |
| 摘要 | 50字以内，说清核心信息 | "新一代大模型推理能力提升10倍，支持多模态理解" |
| 来源和时间 | 发布媒体 + 相对时间 | "36氪 · 2小时前" |
| 原文链接 | **搜索结果中包含文章ID的具体新闻URL** | `https://36kr.com/p/1234567890` |

**重要**：
⚠️ 所有链接必须是搜索结果中的真实新闻来源 URL，严禁编造或使用示例链接
⚠️ 链接必须包含文章ID或唯一标识符，如 `36kr.com/p/1234567890` 而非 `36kr.com/newsflashes`
⚠️ "阅读全文→"链接必须指向新闻报道页面，不是媒体主页、公司官网或标的链接
⚠️ 确保所有链接真实可访问，严禁使用 `#` 或 `example.com`

## 步骤5：生成精美HTML日报

### 生成方式

使用 `generateWebpage` 工具生成最终的HTML早报页面

### 传递给 generateWebpage 的 instruction 要求

**必须逐字传入**前面步骤收集和分析的所有具体数据：
- 新闻标题、摘要、链接
- 投资标的、操作建议、核心逻辑
- 利好/利空催化
**严禁省略任何数据**

### 页面设计规范

#### 整体风格

- **主题**：浅色主题，清新专业，适合晨间阅读
- **参考**：高端财经媒体的视觉语言
- **配色**：
  - 主色：`#4f46e5`（靛蓝）
  - 强调色：`#f59e0b`（琥珀）
  - 危险色：`#ef4444`（红色）
- **字体**：Noto Sans SC（引入 Google Fonts）
- **圆角**：卡片 16px 圆角，柔和阴影
- **动画**：入场动画，卡片依次滑入
- **适配**：移动端完美适配

#### 页面结构（从上到下）

**1. 顶部导航栏**（sticky）
- 左侧：「破晓早报 · M月D日 · 周X」品牌标识，渐变色
- 右侧：板块快捷跳转标签（💰投资 / 📰资讯 / 🍉吃瓜）

**2. 投资观点卡片**（深色渐变背景，视觉焦点）
- 核心观点：大字醒目，关键词高亮（看多用绿色，看空用红色）
- 操作建议表格：标的、方向（买入绿/卖出红/观望蓝标签）、仓位、逻辑
- 利好催化 + 利空预警表格

**3. 风险提示条**（浅红背景，⚠️图标）
- 明确标注「本早报仅供参考，不构成投资建议」

**4. 行业快讯区**
- 按领域分组，每组有领域标题
- 新闻卡片网格布局（响应式，桌面端2-3列，移动端1列）
- 每张卡片：领域标签（彩色小标签）、标题、摘要、来源时间、「阅读全文→」链接
- 卡片 hover 有上浮阴影效果

**5. 吃瓜醒脑区**
- 粉色左边框的卡片
- 热度🔥标识 + 事件标题 + 犀利点评（斜体灰底）

**6. 底部**
- 日期
- 免责声明

#### 关键要求

⚠️ 所有 `href` 链接必须使用搜索结果中的真实新闻 URL，严禁使用 `#`、`example.com` 或示例链接
⚠️ **链接必须包含文章ID**：`https://36kr.com/p/1234567890` ✅ 正确 vs `https://36kr.com/newsflashes` ❌ 错误
⚠️ 日期使用当天真实日期
⚠️ 时间标注使用「X 小时前」「昨夜」「今晨」等相对时间
⚠️ 引入 Font Awesome 图标库
⚠️ **重点检查**：每条新闻的"阅读全文→"链接必须对应搜索到的新闻来源链接，不是公司官网或股票标的链接，更不是媒体主页
