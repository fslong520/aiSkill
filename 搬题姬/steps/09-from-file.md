# Step 9: 自文件取题面

## 目标

自用户所供本地文件（如 .md、.txt、截图等）取题目信息，
并按比赛搬运流程（contest/）逐题生成完整题包。

## 支持文件类型

| 类型 | 处理方式 |
|------|---------|
| .md / .txt | 直接 `read_file` 读取 |
| 图片 (.png/.jpg) | OCR 或视觉模型提取文字 |
| HTML | 解析 DOM 提取题面 |

## 重要变更

本步骤**不再走独立单题流程**，而是统一走 **比赛搬运流程（steps/contest/）**，
即使只有一道题也按"1 道题的比赛"处理。

这样确保：
- 单文件和多文件搬运行为一致
- 题面汇总、逐题打包、合集打包的统一处理
- 避免单题流程与比赛流程的格式差异

## 步骤

### 1. 收集文件清单

确认用户提供的所有文件路径，可能是一个或多个：

```bash
# 单个文件
files=("/path/to/problem.md")

# 多个文件
files=("/path/to/a.md" "/path/to/b.md" "/path/to/c.md")
```

### 2. 创建比赛 ID

```bash
# 单个文件：用文件名前缀作为比赛 ID
# 多个文件：用共同父目录名作为比赛 ID，或询问用户
contest_id="contest_$(date +%Y%m%d)"
```

### 3. 创建题面汇总文件

仿照 `steps/contest/01-list.md`，生成题面汇总文件 `{contest_id}.md`：

```markdown
<div class="water">

# {contest_id}

---

（逐题追加写入）

</div>
```

### 4. 逐题读取并写入汇总文件

对每个文件：
1. 读取文件内容
2. 提取题名、题述、输入输出格式、样例、数据范围
3. 按 `steps/contest/02-problem.md` 格式翻译并追加写入 `{contest_id}.md`

### 5. 转入比赛搬运流程

```bash
# 现在题面汇总文件已就绪，转交 contest/03-move.md 逐题打包
```

## 关键区别

| 维度 | 旧流程 | 新流程 |
|:-----|:-------|:-------|
| 单文件 | 独立单题流程 | 走比赛流程（1题比赛）|
| 多文件 | 无法处理 | 走比赛流程（N题比赛）|
| 题面汇总 | 不生成 | 生成 `{contest_id}.md` |
| 打包 | 单题 zip | 合集 zip（work_* 在根）|
