# Step 3: GESP 知识点等级判定

## 目标

定题目知识点等级（GESP 1-8级）与难度等级（二者独立评估）。

## 原则

知识点等级 ≠ 题目难度。同知识点可出简题，亦可出难题。

**先定级，后定难**：先按所涉知识点定GESP级别，后独立评估题目难度。

---

## 第一步：前置分析（必做）

在判定等级前，先提取题目特征：

### 1.1 输入输出特征

| 特征 | 信息 | 提示 |
|------|------|------|
| 输入规模 | N 的最大值 | 判断算法复杂度要求 |
| 输出类型 | 单值/序列/构造 | 影响解法选择 |
| 多组数据 | T 的存在 | 可能涉及预处理 |

### 1.2 数据范围速判

| N 的范围 | 允许复杂度 | 常用算法 |
|----------|-----------|---------|
| N ≤ 20 | O(2^N) | 暴力枚举、位运算 |
| N ≤ 1000 | O(N^2) | 暴力DP、简单图论 |
| N ≤ 10^5 | O(N log N) | 二分、排序、线段树 |
| N ≤ 10^6 | O(N) | 贪心、前缀和、并查集 |
| N ≤ 10^9 | O(log N) 或 O(1) | 数学公式、矩阵快速幂 |

### 1.3 空间限制

| 限制 | 提示 |
|------|------|
| 256MB | 可开二维数组 |
| 64MB | 注意空间优化 |
| 16MB | 需滚动数组/压缩 |

## 第二步：算法识别（核心）

### 2.1 算法分类详表

#### 基础算法（1-4级）

| 级别 | 算法 | 典型特征 | 代码标志 |
|:----:|------|---------|---------|
| 一级 | 顺序执行 | 无分支，直接计算 | 仅 cin/cout + 简单运算 |
| 一级 | 单分支 | 一个 if-else | if (a > b) ... |
| 一级 | 单层循环 | 一个 for/while | for (i=0; i<n; i++) |
| 二级 | 多层嵌套 | 2-3层循环 | for { for { } } |
| 二级 | 多分支 | if-else if-else 链 | 多个条件判断 |
| 三级 | 枚举 | 暴力尝试所有可能 | 嵌套循环 + 判断 |
| 三级 | 模拟 | 按题意逐步实现 | 无特殊算法，直接翻译 |
| 四级 | 递推 | 状态转移 f(n) = f(n-1) + ... | dp[i] = dp[i-1] + ... |
| 四级 | 冒泡排序 | 相邻比较交换 | for { for { if > swap } } |
| 四级 | 插入排序 | 逐步插入有序序列 | for { while { } } |
| 四级 | 选择排序 | 每轮选最小 | for { min = i; for { } } |

#### 中级算法（5-6级）

| 级别 | 算法 | 典型特征 | 代码标志 |
|:----:|------|---------|---------|
| 五级 | 二分查找 | 有序数组中查找 | while (l <= r) { mid = ... } |
| 五级 | 二分答案 | 答案在某范围内 | check() 函数 + 二分 |
| 五级 | 贪心 | 局部最优 → 全局最优 | sort + 依次选取 |
| 五级 | 分治 | 问题分解为子问题 | 递归 + 合并 |
| 五级 | 快速排序 | 分治排序 | partition + 递归 |
| 五级 | 归并排序 | 分治合并 | merge + 递归 |
| 五级 | 递归 | 函数调用自身 | void f() { f(); } |
| 六级 | DFS | 深度优先搜索 | void dfs() { for adj } |
| 六级 | BFS | 广度优先搜索 | queue + while |
| 六级 | 简单DP | 一维/二维状态 | dp[i] = max/min(...) |
| 六级 | 背包DP | 01/完全/多重背包 | for { for { dp[j] = ... } } |
| 六级 | 树遍历 | 前/中/后序遍历 | 递归遍历左右子树 |
| 六级 | 栈应用 | 括号匹配、表达式 | stack + push/pop |
| 六级 | 队列应用 | BFS、滑动窗口 | queue + push/pop |

#### 高级算法（7-8级）

| 级别 | 算法 | 典型特征 | 代码标志 |
|:----:|------|---------|---------|
| 七级 | 复杂DP | 多维/区间/树形DP | dp[i][j] = ... |
| 七级 | LIS/LCS | 最长递增/公共子序列 | 二分优化 or dp[i][j] |
| 七级 | 并查集 | 连通性、合并集合 | find() + union() |
| 七级 | 拓扑排序 | 有向无环图排序 | 入度 + queue |
| 七级 | 图遍历 | 邻接表/矩阵遍历 | for adj[u] |
| 七级 | 哈希表 | 快速查找 | unordered_map/set |
| 八级 | Dijkstra | 最短路 | priority_queue + 松弛 |
| 八级 | Bellman-Ford | 负权最短路 | for { for { relax } } |
| 八级 | Floyd | 全源最短路 | for k { for i { for j } } |
| 八级 | Kruskal | 最小生成树 | sort + 并查集 |
| 八级 | Prim | 最小生成树 | priority_queue |
| 八级 | 倍增 | LCA、快速幂 | dp[i][j] = dp[i-1][...] |
| 八级 | 线段树 | 区间查询/修改 | build + query + update |
| 八级 | 树状数组 | 区间求和 | lowbit + 前缀和 |

### 2.2 特殊算法识别

| 算法 | 级别 | 识别方法 |
|------|:----:|---------|
| 双指针/尺取 | 五级 | 两个变量同步移动 |
| 单调栈/队列 | 七级 | 维护单调性 |
| CDQ分治 | 八级 | 三维偏序 |
| 网络流 | 八级 | 最大流/最小割 |
| 后缀数组/自动机 | 八级 | 字符串后缀处理 |
| 平衡树 | 八级 | 有序集合操作 |

## 第三步：定级（综合判定）

### 3.1 核心规则

1. **算法优先**：以最高级别算法为准定级
2. **数据结构不独立定级**：数据结构是算法的容器，不单独决定等级
3. **数学不计级**：组合数学、数论等理解性知识不计入定级
4. **模板题降级**：若只需套模板，可酌情降一级

### 3.2 组合判定表

| 组合情况 | 定级规则 |
|---------|---------|
| 算法A + 算法B | 取 max(级别A, 级别B) |
| 算法 + 数据结构 | 按算法定级 |
| 算法 + 数学 | 按算法定级 |
| 多算法串联 | 取最高级算法 |
| 多算法并行 | 取最高级算法 |

### 3.3 等级边界判定

| 情况 | 判定 |
|------|------|
| 一级 vs 二级 | 有无嵌套循环 |
| 二级 vs 三级 | 有无数组/字符串操作 |
| 三级 vs 四级 | 有无递推/排序 |
| 四级 vs 五级 | 有无二分/贪心/递归 |
| 五级 vs 六级 | 有无DFS/BFS/DP |
| 六级 vs 七级 | 有无复杂DP/并查集/拓扑 |
| 七级 vs 八级 | 有无最短路/MST/倍增/线段树 |

## 第四步：检查清单（必做）

### 4.1 自查清单

完成定级后，逐项检查：

- [ ] **算法识别完整**：是否遗漏了题目中的算法？
- [ ] **等级取最高**：是否按最高级算法定级？
- [ ] **数据结构不独立**：是否错误地用数据结构定级？
- [ ] **数学不计级**：是否错误地将数学知识计入等级？
- [ ] **边界情况**：是否处于等级边界？需要仔细判断。
- [ ] **常见误判**：是否犯了常见误判中的错误？

### 4.2 常见误判速查

| 误判 | 正确 | 原因 |
|------|------|------|
| 尺取法 + 字符串 → 三级 | **五级** | 算法优先：尺取法是五级 |
| 二分 + 数组 → 三级 | **五级** | 算法优先：二分是五级 |
| DFS + 二维数组 → 四级 | **六级** | 算法优先：DFS是六级 |
| 递推 + 一维数组 → 三级 | **四级** | 算法优先：递推是四级 |
| 并查集 → 三级 | **七级** | 并查集是七级图算法 |
| BFS + 简单图 → 六级 | **六级** | BFS是六级，不是七级 |
| Dijkstra → 七级 | **八级** | 最短路是八级 |
| 线段树 → 七级 | **八级** | 线段树是八级数据结构 |

### 4.3 复核流程

```
1. 识别题目中所有算法
2. 查表确定每个算法的级别
3. 取最高级别作为初步定级
4. 检查是否有"数学不计级"的情况
5. 检查是否处于等级边界
6. 对照常见误判表检查
7. 输出最终定级
```

## 第五步：定难（思维复杂度）

**独立于GESP级别**：一级题可甚难（复杂嵌套），八级题可甚简（直套模板）。

### 难度等级（五星制）

| 星级 | 特征 | 举例 |
|:----:|------|------|
| ⭐ | 直套公式/模板，一步到位 | 直分支判断、数组逆序、简公式 |
| ⭐⭐ | 需简变形或组合2知识点 | 结构体排序、多条件筛选 |
| ⭐⭐⭐ | 需算法综合或思维转化 | DP态设计、图遍历变体 |
| ⭐⭐⭐⭐ | 复杂优化、数据结构结合 | 线段树优化、复杂图论 |
| ⭐⭐⭐⭐⭐ | 高难竞赛题 | 高级算法组合 |

### 定难规则

1. **看拐点**：需"想一步"否，而非"直做"？
2. **看组合**：需组多不相关知识点否？
3. **看边界**：正理边界需特殊思维否？
4. **看数据**：需算法效率上动脑否？

## 2026大纲要点

1. **数组限制**：三级/四级数组不含变长数组（vector）
2. **函数限制**：四级函数不含Lambda/运算符重载
3. **C++标准**：以C++11为准
4. **复杂度**：需分析时间/空间复杂度
