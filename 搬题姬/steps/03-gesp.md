# Step 3: GESP 知识点等级判定

## 目标

定题目知识点等级（GESP 1-8级）与难度等级（二者独立评估）。

## 原则

知识点等级 ≠ 题目难度。同知识点可出简题，亦可出难题。

**先定级，后定难**：先按所涉知识点定GESP级别，后独立评估题目难度。

---

## 第一步：前置分析（必做）

在判定等级前，先提取题目特征：

### 1.1 输入输出特征

| 特征 | 信息 | 提示 |
|------|------|------|
| 输入规模 | N 的最大值 | 判断算法复杂度要求 |
| 输出类型 | 单值/序列/构造 | 影响解法选择 |
| 多组数据 | T 的存在 | 可能涉及预处理 |

### 1.2 数据范围速判

| N 的范围 | 允许复杂度 | 常用算法 |
|----------|-----------|---------|
| N ≤ 20 | O(2^N) | 暴力枚举、位运算 |
| N ≤ 1000 | O(N^2) | 暴力DP、简单图论 |
| N ≤ 10^5 | O(N log N) | 二分、排序、线段树 |
| N ≤ 10^6 | O(N) | 贪心、前缀和、并查集 |
| N ≤ 10^9 | O(log N) 或 O(1) | 数学公式、矩阵快速幂 |

### 1.3 空间限制

| 限制 | 提示 |
|------|------|
| 256MB | 可开二维数组 |
| 64MB | 注意空间优化 |
| 16MB | 需滚动数组/压缩 |

## 第二步：大纲知识点识别

依据 CCF GESP 官方大纲（2026版）将题目所涉知识点归入对应级别。

### 2.1 核心知识总览

| 级别 | 核心知识 | 定级提示 |
|:----:|---------|---------|
| 一级 | 变量、基本数据类型、顺序/分支/循环、基本运算、输入输出 | 无数组、无嵌套循环，仅有基础语法 |
| 二级 | 存储与网络、流程图、ASCII编码、数据类型转换、**多层分支/循环嵌套**、数学函数 | 有嵌套循环，但无数组/字符串操作 |
| 三级 | 数据编码、进制转换、位运算、算法描述、**一维数组、字符串、枚举法、模拟法** | 用到一维数组或字符串，或枚举/模拟算法 |
| 四级 | **指针、结构体、二维/多维数组、函数、递推算法、排序算法（冒泡/插入/选择）、文件操作、异常处理** | 用到二维数组/结构体/指针，或递推/排序算法 |
| 五级 | **初等数论、高精度运算、链表、二分查找/答案、递归、贪心、分治**（归并/快排） | 用到二分/贪心/递归/分治/数论/链表 |
| 六级 | **树、哈夫曼树、二叉排序树、DFS/BFS、简单动态规划、面向对象、栈与队列** | 用到树/图遍历/DFS/BFS/简单DP/栈/队列 |
| 七级 | 数学库函数、**复杂动态规划**（LIS/LCS/区间DP）、**图的定义与遍历、图论算法、哈希表** | 用到复杂DP、图论、哈希表 |
| 八级 | 计数原理、排列组合、杨辉三角、**倍增法、图论算法综合**（MST/最短路）、算法优化 | 用到MST/最短路/倍增/组合数学/算法优化 |

### 2.2 核心算法递进

此表来自大纲附录。**判定时以题目所涉最高级算法为准。**

| 级别 | 核心算法 |
|:----:|---------|
| 一级 | 无特定算法，基础编程 |
| 二级 | 无特定算法，综合编程 |
| 三级 | 枚举法、模拟法 |
| 四级 | 递推算法、冒泡排序、插入排序、选择排序 |
| 五级 | 二分查找/二分答案、递归、贪心算法、分治（归并排序、快速排序） |
| 六级 | 深度优先搜索（DFS）、宽度优先搜索（BFS）、简单动态规划 |
| 七级 | 复杂动态规划（LIS、LCS、区间DP）、图遍历算法 |
| 八级 | 最小生成树、最短路径、算法优化 |

### 2.3 核心数据结构递进

此表来自大纲附录。**若题目使用某级数据结构，其定级不可低于该级。**

| 级别 | 核心数据结构 | 说明 |
|:----:|-------------|------|
| 一级 | 基本数据类型（int、long long、float、double、char、bool） | 无复合结构 |
| 二级 | 无新增 | 仅有基本类型 |
| 三级 | **一维数组、字符串** | 用一维数组/字符串解题→下限三级 |
| 四级 | **二维/多维数组、结构体、指针** | 用二维数组/结构体/指针→下限四级 |
| 五级 | 链表（单链表、双链表、循环链表） | 用链表→下限五级 |
| 六级 | 栈、队列、循环队列、树、二叉树 | 用栈/队列/树→下限六级 |
| 七级 | 图（有向图、无向图）、哈希表 | 用图/哈希表→下限七级 |
| 八级 | 无新增（综合应用） | 综合运用已有知识 |

## 第三步：定级（综合判定）

### 3.1 核心规则

1. **算法优先**：以最高级别算法为准定级（查 2.2 节）
2. **数据结构设下限**：题目所需数据结构决定最低等级（查 2.3 节），算法等级若高于此则取算法
3. **数学不计级**：组合数学、数论等理解性知识不计入定级
4. **模板题降级**：若只需套模板，可酌情降一级

### 3.2 组合判定表

| 组合情况 | 定级规则 |
|---------|---------|
| 算法A + 算法B | 取 max(级别A, 级别B) |
| 算法 + 数据结构 | 取 max(算法级别, 数据结构最低级别) |
| 算法 + 数学 | 按算法定级 |
| 多算法串联 | 取最高级算法 |
| 多算法并行 | 取最高级算法 |

### 3.3 等级边界判定

| 情况 | 判定 |
|------|------|
| 一级 vs 二级 | 有无嵌套循环 |
| 二级 vs 三级 | 有无数组/字符串操作 |
| 三级 vs 四级 | 有无递推/排序/二维数组/结构体 |
| 四级 vs 五级 | 有无二分/贪心/递归/链表 |
| 五级 vs 六级 | 有无DFS/BFS/DP/树/栈/队列 |
| 六级 vs 七级 | 有无复杂DP/图遍历/哈希表 |
| 七级 vs 八级 | 有无MST/最短路/倍增 |

### 3.4 定级示例

| 题特征 | 最高算法 | 数据结构 | 数据结构下限 | 最终级别 |
|-------|:-------:|:-------:|:----------:|:-------:|
| A+B | 无 | 基本类型 | 一级 | **一级** |
| 判断闰年，多分支 | 无 | 基本类型 | 一级 | **一级** |
| 求1到N和，单层循环 | 无 | 基本类型 | 一级 | **一级** |
| 打印九九表，嵌套循环 | 无 | 基本类型 | 一级 | **一级**（无数组） |
| 数组最大值 | 无 | 一维数组 | 三级 | **三级** |
| 字符串反转 | 无 | 字符串 | 三级 | **三级** |
| 暴力枚举两数之和 | 枚举（三级） | 一维数组 | 三级 | **三级** |
| 模拟井字棋 | 模拟（三级） | 二维数组 | 四级 | **四级** |
| 雷域统计（二维数组遍历） | 无特定算法 | 二维数组 | 四级 | **四级** |
| 结构体排序 | 排序（四级） | 结构体 | 四级 | **四级** |
| 二分查找 | 二分（五级） | 一维数组 | 三级 | **五级**（算法>数据结构） |
| 走迷宫DFS | DFS（六级） | 二维数组 | 四级 | **六级**（算法>数据结构） |
| Dijkstra最短路 | 最短路（八级） | 图 | 七级 | **八级**（算法>数据结构） |

## 第四步：检查清单（必做）

### 4.1 自查清单

完成定级后，逐项检查：

- [ ] **2.1 核心知识**：题目知识点覆盖哪一级核心知识？
- [ ] **2.2 算法识别**：最高级算法查表后对应哪一级？
- [ ] **2.3 数据结构下限**：所用数据结构设定了什么下限？
- [ ] **3.1 综合判定**：取 max(算法级别, 数据结构下限) 后结果为何？
- [ ] **数学不计级**：是否错误地将数学知识计入等级？
- [ ] **边界情况**：是否处于等级边界？仔细判断。
- [ ] **常见误判**：是否犯了常见误判中的错误？

### 4.2 常见误判速查

| 误判 | 正确 | 原因 |
|------|------|------|
| 求1到N和 → 一级 | **一级** ✅ | 单层循环，无数组，一级正确 |
| 数组求最大最小值 → 二级 | **三级** | 一维数组是三级知识，数据结构下限三级 |
| 二维数组遍历 → 三级 | **四级** | 二维数组是四级知识，数据结构下限四级 |
| 结构体排序 → 三级 | **四级** | 结构体+排序均属四级 |
| 二分查找+数组 → 三级 | **五级** | 算法优先：二分查找是五级 |
| 递推+一维数组 → 三级 | **四级** | 算法优先：递推是四级 |
| DFS走迷宫 → 四级 | **六级** | 算法优先：DFS是六级 |
| 并查集 → 三级 | **七级** | 并查集是七级图算法 |
| Dijkstra → 七级 | **八级** | 最短路是八级 |

### 4.3 复核流程

```
1. 识别题目的核心知识 → 查 2.1 节初步定位
2. 识别题目的最高级算法 → 查 2.2 节得算法级别
3. 识别题目的最高级数据结构 → 查 2.3 节得数据结构下限
4. 取 max(算法级别, 数据结构下限) → 初步定级
5. 检查是否有"数学不计级"或"模板可降级"情况
6. 对照 3.4 节示例和 4.2 节常见误判确认
7. 输出最终定级
```

## 第五步：定难（思维复杂度）

**独立于GESP级别**：一级题可甚难（复杂嵌套），八级题可甚简（直套模板）。

### 难度等级（五星制）

| 星级 | 特征 | 举例 |
|:----:|------|------|
| ⭐ | 直套公式/模板，一步到位 | 直分支判断、数组逆序、简公式 |
| ⭐⭐ | 需简变形或组合2知识点 | 结构体排序、多条件筛选 |
| ⭐⭐⭐ | 需算法综合或思维转化 | DP态设计、图遍历变体 |
| ⭐⭐⭐⭐ | 复杂优化、数据结构结合 | 线段树优化、复杂图论 |
| ⭐⭐⭐⭐⭐ | 高难竞赛题 | 高级算法组合 |

### 定难规则

1. **看拐点**：需"想一步"否，而非"直做"？
2. **看组合**：需组多不相关知识点否？
3. **看边界**：正理边界需特殊思维否？
4. **看数据**：需算法效率上动脑否？

## 第六步：定分（CF Rating 估算）

**目标**：为题目估算 Codeforces Rating（800~3500），写入 `problem.yaml` 的 `score` 字段。
此步在定级（第三步）和定难（第五步）完成之后执行，综合二者结果。

### 6.1 基分表（GESP 等级 → 基准 Rating）

> ⚠️ 本表基于 Codeforces 实际题目 Rating 统计校准，非理论推算。
> 例如：CF 上纯 0/1 背包模板题 ≈ 1300，树形 DP ≈ 1600，Dijkstra ≈ 1700。

| GESP级别 | 基准分 | 典型 CF Rating 区间 | CF 参考题 |
|:--------:|:-----:|:-------------------:|-----------|
| 一级 | 800 | 800~900 | CF 4A (800) |
| 二级 | 900 | 900~1000 | CF 282A (900) |
| 三级 | 1050 | 1000~1200 | CF 263A (1100) |
| 四级 | 1200 | 1100~1300 | CF 69A (1200) |
| 五级 | 1300 | 1200~1400 | CF 189A Cut Ribbon (1300, DP) |
| **六级** | **1350** | **1200~1500** | CF 1446A Knapsack (1300, 贪心背包) |
| **七级** | **1600** | **1500~1900** | CF 698A (1500, DP), CF 474D (1700, DP) |
| **八级** | **2000** | **1900~2500** | CF 1132E Knapsack (2100), CF 166E (2200, DP) |

> **校准依据**：CF 背包类题如 1446A(Knapsack) = 1300，复杂背包+剪枝如 1132E = 2100。
> 简单 DP（斐波那契/递推）≈ 1200，标准背包模板 ≈ 1300~1400，
> 背包 + 变形/优化 ≈ 1500~1700，高级 DP（状压/树形）≈ 1800+。

**取法**：以 GESP 等级查基准分，再以下列因素上下浮动。

### 6.2 浮动修正

#### 难度星级修正

| 难度星级 | 修正值 | 说明 |
|:--------:|:------:|------|
| ⭐ | 基准 -100 | 模板/直套，降一档 |
| ⭐⭐ | ±0 | 正常难度 |
| ⭐⭐⭐ | +100 | 需思维转化 |
| ⭐⭐⭐⭐ | +200 | 复杂优化 |
| ⭐⭐⭐⭐⭐ | +300+ | 高难竞赛 |

#### 数据规模修正

题目 n 的范围暗示所需时间复杂度，可辅判 Rating：

| 规模特征 | 预期复杂度 | 修正倾向 |
|---------|-----------|---------|
| n ≤ 20 | 指数级（状压/暴搜） | rating 常 ≥ 1600 |
| n ≤ 10^5 | O(n log n) | 主流题，1200~2000 |
| n ≤ 10^9 | O(log n) 或数学 | 可低可高，看思维量 |

#### 已知 CF Rating（最高优先级）

若题目来自 Codeforces 且已知 Rating：
- **直接采用已知 Rating**，跳过估算
- 例：CF Round 某题 Rating 1400 → `score: 1400`

#### AtCoder → CF 难度换算

AtCoder 题目难度分（题面所标 Difficulty）不可径用——须换算为 CF Rating 方可填入 `score`。

| 优先级 | 方法 | 条件 |
|:------:|------|------|
| 最高 | Difficulty 分数换算 | 题面可见 AtCoder Difficulty 值 |
| 次高 | 比赛位置估分 | 无 Difficulty 时，按比赛+题号估算 |
| 最低 | GESP 基分法 | 两者皆无，回退至 GESP 估分 |

##### 方法一：Difficulty 分数换算（优先）

AtCoder 题面标有 Difficulty 值（如 `Difficulty: 1287`），按下表换算为 CF Rating。

| AtCoder Difficulty | CF Rating 区间 | 取中值（快速参考） |
|:------------------:|:--------------:|:------------------:|
| 0~99 | 800 | 800 |
| 100~399 | 800~900 | 850 |
| 400~799 | 900~1100 | 1000 |
| 800~1199 | 1100~1400 | 1250 |
| 1200~1599 | 1400~1700 | 1550 |
| 1600~1999 | 1700~2000 | 1850 |
| 2000~2399 | 2000~2300 | 2150 |
| 2400~2799 | 2300~2700 | 2500 |
| 2800~3199 | 2700~3100 | 2900 |
| 3200+ | 3100~3500 | 3300 |

**换算公式**（同区间内插值用）：

```
diff ≤ 100:  CF = 800
100 < diff ≤ 400:  CF = 800 + (diff - 100) × 0.5
400 < diff ≤ 1200: CF = 950 + (diff - 400) × 0.375
1200 < diff ≤ 2000: CF = 1250 + (diff - 1200) × 0.75
2000 < diff ≤ 2800: CF = 1850 + (diff - 2000) × 1.0
2800 < diff: CF = 2650 + (diff - 2800) × 1.0, cap 3500
```

例：AtCoder Difficulty 1287
→ 查表 1200~1599 区间 → CF 1400~1700
→ 公式：1250 + (1287 - 1200) × 0.75 = 1250 + 65 = 1315 → 取整 1300

##### 方法二：比赛位置估分（Difficulty 不可知时）

| 比赛 | 题号 | CF 近似 |
|:----:|:----:|:-------:|
| ABC | A | 800~900 |
| ABC | B | 900~1100 |
| ABC | C | 1100~1300 |
| ABC | D | 1300~1600 |
| ABC | E | 1600~2000 |
| ABC | F | 1900~2400 |
| ABC | G（旧ARC） | 2200~2800 |
| ARC | A(C) | 1600~2100 |
| ARC | B(D) | 2000~2500 |
| ARC | C(E) | 2400~2900 |
| AGC | A | 2000~2800 |
| AGC | B+ | 2800~3500 |

**注意**：比赛位置法较粗糙，同一位置因场次不同可 ±200 偏差。有 Difficulty 值时务必用方法一。

### 6.3 估分公式

```
score = 基准分 + 难度修正 + 数据规模修正
```

- 结果须落在 800~3500 区间
- 取整至 50 的倍数（如 1300、1450、1700）
