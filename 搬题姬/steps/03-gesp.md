# Step 3: GESP 知识点等级判定

## 目标

判定题目知识点等级（GESP 1-8级）和难度等级（两者独立评估）。

## 原则

知识点等级 ≠ 题目难度。同一知识点可出简单题，也可出难题。

**先定级，后定难**：先按涉及知识点确定GESP级别，再独立评估题目难度。

---

## 第一步：定级（知识点范围）

### 等级表（2026年最新大纲）

| 级别 | 核心算法 | 数据结构 | 数学/其他 |
|------|---------|----------|----------|
| 一级 | 无（顺序/分支/循环） | 基本类型 | 计算机基础、ASCII |
| 二级 | 枚举、模拟 | 无新增 | 数学函数、类型转换 |
| 三级 | 枚举、模拟 | 一维数组（不含变长）、字符串 | 进制转换、位运算 |
| 四级 | 递推、排序（冒泡/插入/选择） | 二维数组、结构体、指针基础 | 函数、文件操作 |
| 五级 | 递归、分治（快排/归并）、贪心、二分 | 链表（单/双/循环） | 高精度、数论（GCD/素数筛） |
| 六级 | DFS、BFS、简单DP | 树（遍历/哈夫曼）、栈、队列 | 面向对象基础 |
| 七级 | 复杂DP、图遍历 | 图（邻接矩阵/表）、哈希表 | 数学库函数 |
| 八级 | 最小生成树、最短路、倍增 | 无新增 | 组合数学、复杂度分析 |

### 定级规则

1. **取最高**：涉及多知识点时，按最高级别定级
2. **算法优先**：算法决定级别上限，数据结构辅助判断
3. **只看考纲**：不管题目实际难度，只看涉及哪些考纲知识点

### 快速定级

```
【数据结构路径】
基本类型     → 一级
数组/字符串  → 三级（注意：不含vector等变长数组）
二维数组      → 四级
结构体/指针   → 四级
链表          → 五级
树/图         → 六～七级
哈希表        → 七级

【算法路径】
顺序/分支/循环 → 一级
枚举/模拟      → 二级
数组遍历/统计  → 三级
递推/排序      → 四级
二分/贪心/分治 → 五级
搜索(DFS/BFS) → 六级
动态规划       → 六～七级
图论算法       → 八级
```

### 常见题目对应

| 题目特征 | GESP级别 |
|---------|----------|
| A+B、判断奇偶、计算器、公式计算 | 一级 |
| 简单枚举、数学函数应用 | 二级 |
| 进制转换、位运算、数组统计、逆序 | 三级 |
| 二维数组、结构体、排序、递推 | 四级 |
| 高精度、链表、二分答案、GCD | 五级 |
| 树遍历、迷宫(DFS/BFS)、背包DP | 六级 |
| LIS/LCS、图遍历、复杂DP | 七级 |
| 最短路、最小生成树、组合数学 | 八级 |

## 第二步：定难（思维复杂度）

**独立于GESP级别**：一级题可以很难（复杂嵌套），八级题可以简单（直接套模板）。

### 难度等级

| 难度 | 特征 | 举例 |
|------|------|------|
| 入门 | 直接套公式/模板，一步到位 | 直接分支判断、数组逆序、简单公式 |
| 普及- | 需简单变形或组合2个知识点 | 结构体排序、多条件筛选 |
| 普及/提高- | 需算法综合或思维转化 | DP状态设计、图遍历变体 |
| 提高/省选- | 复杂优化、数据结构结合 | 线段树优化、复杂图论 |
| 省选/NOI | 高难度竞赛题 | 高级算法组合 |

### 定难规则

1. **看拐点**：题目是否需要"想一步"而不是"直接做"？
2. **看组合**：是否需要组合多个不相关的知识点？
3. **看边界**：是否正确处理边界需要特殊思维？
4. **看数据**：是否需要在算法效率上动脑筋？

### 常见误判

| 误判 | 例子 | 正确 |
|-----|------|------|
| 用了数组就是三级难度 | 成绩倒序（一级知识点+一级难度） | GESP三级、难度入门 |
| 用了排序就是四级难度 | 冒泡排序给5个数排序 | GESP四级、难度入门 |
| 一级知识点只能是入门 | 复杂嵌套条件+多种特殊情况 | GESP一级、难度普及- |

## 2026大纲重要变更

1. **明确排除超纲内容**：三级/四级数组不含变长数组（vector）；四级函数不含Lambda/运算符重载
2. **C++标准统一**：所有考试以C++11为准
3. **强调效率意识**：需分析时间和空间复杂度
