## T1 - 扫雷统计
Minesweeper

</center>

### 题目描述
给定一个 n 行 m 列的地图，其中有 q 个雷区。

对于每个格子：
- 如果它是雷区，输出 `*`；
- 如果不是雷区，输出它周围 8 个方向中有多少个雷区。

两个格子只要有公共边或公共顶点，就算相邻。

### 输入格式
第一行三个整数 n, m, q（1 ≤ n, m ≤ 500，1 ≤ q ≤ n×m），表示地图的行数、列数和雷区数量。

接下来 q 行，每行两个整数 x, y（1 ≤ x ≤ n，1 ≤ y ≤ m），表示一个雷区的坐标。保证所有雷区坐标互不相同。

### 输出格式
输出 n 行，每行 m 个用空格隔开的字符或数字：
- 如果 (i, j) 是雷区，输出 `*`；
- 否则输出 (i, j) 周围 8 个方向中雷区的数量。

### 样例输入 #1
```
3 3 2
1 1
3 3
```

### 样例输出 #1
```
* 1 0
1 2 1
0 1 *
```

### 数据范围
1 ≤ n, m ≤ 500
1 ≤ q ≤ n×m
1 ≤ x ≤ n，1 ≤ y ≤ m

**知识点**：GESP四级（二维数组、枚举/模拟） | 难度：⭐⭐

## T2 - BMI 排序
BMI Sort

</center>

### 题目描述
有 n 个小朋友，第 i 个小朋友的编号是 i。

已知：
- 体重 w_i，单位 kg；
- 身高 h_i，单位 m。

身高体重指数 BMI 定义为：BMI = w / h²。

要求按照 BMI 从高到低排序，输出排序后的编号。

### 输入格式
第一行一个整数 n（1 ≤ n ≤ 1000），表示小朋友的数量。

第二行 n 个整数 w_1, w_2, ..., w_n，表示每个小朋友的体重。

第三行 n 个浮点数 h_1, h_2, ..., h_n，表示每个小朋友的身高（保留两位小数）。

### 输出格式
输出一行，n 个用空格隔开的整数，表示按 BMI 从高到低排序后的编号。

如果 BMI 相同，则编号小的优先。

### 样例输入 #1
```
4
60 80 50 70
1.50 1.60 1.40 1.50
```

### 样例输出 #1
```
2 4 1 3
```

### 样例解释 #1
各小朋友的 BMI 如下（保留两位小数）：
1号：60 / 1.50² = 26.67
2号：80 / 1.60² = 31.25
3号：50 / 1.40² = 25.51
4号：70 / 1.50² = 31.11

按 BMI 降序排列：2号(31.25) > 4号(31.11) > 1号(26.67) > 3号(25.51)。

### 数据范围
1 ≤ n ≤ 1000
1 ≤ w_i ≤ 200
1.00 ≤ h_i ≤ 2.00（保留两位小数）

**知识点**：GESP四级（结构体、排序、浮点数处理） | 难度：⭐⭐
