Summary: 分析素材→问答引导→鲁迅风格创作，生成2000-3000字批判性文章。

Strategy:
1. 素材分析（提取矛盾、现象、人群、根源）
2. 动态生成问题（批判焦点、目标读者、期望反应）
3. 确认文章框架和标题
4. 鲁迅风格创作（反问、对比、讽刺）
5. 添加配图提示（4-6处）

AVOID:
- AVOID 套用固定问题模板，要根据素材动态生成
- AVOID 直接暴露考点，要包装成故事

- AVOID 人身攻击或恶意诽谤
- AVOID 煽动仇恨或歧视

## 工作流程

### Phase 1: 素材分析

提取要素：
- **核心矛盾**：最突出的冲突或反差
- **关键现象**：值得批判的具体表现
- **涉及人群**：哪些人被影响
- **可能根源**：表面背后隐藏什么

判断文章类型：社会现象批判 / 文化反思 / 人性剖析 / 制度批评

### Phase 2: 问答引导

**核心原则：根据素材内容动态生成选择项，绝不套用固定模板**

#### 问题设计公式

| 问题 | 选项来源 |
|------|---------|
| 批判焦点 | 从素材提取4个最明显的矛盾点 |
| 目标读者 | 根据素材涉及的人群定制 |
| 期望反应 | 根据批判焦点设计 |

**示例（AI工具炒作素材）**：
```
问题1：批判焦点
- A：盲目追捧AI工具的浮躁风气
- B：程序员群体的焦虑与自我怀疑
- C：AI行业的泡沫与虚假宣传
- D：技术与人性关系的异化现象
```

**第二轮追问**（根据选择触发）：
- 选行业泡沫→追问资本运作/媒体推波/从业者自欺/监管缺失
- 选焦虑相关→追问技术迭代/同辈压力/不确定性/价值认同

### Phase 3: 框架确认

生成文章框架供用户确认：

```markdown
## 拟定标题
- A：尖锐直接型（一针见血）
- B：反讽设问型（含蓄有力）
- C：对比冲突型（视觉感强）

## 文章结构（2000-3000字）

**开头（200-300字）**：从素材中最有代表性的场景切入
**主体论证1（500-700字）**：剖析现象层面，列举2-3个案例
**主体论证2（500-700字）**：深入本质，分析根源
**主体论证3（400-600字）**：扩展延伸，揭示普遍性
**案例分析（300-400字）**：深度剖析最具代表性的案例
**结尾升华（200-300字）**：上升到深层主题，留有余味

## 核心论点
[一句话批判观点]

## 情感基调
[愤怒/嘲讽/悲哀/无奈/复杂混合]
```

### Phase 4: 文章创作

#### 鲁迅风格要素

| 手法 | 示例 |
|------|------|
| 反问 | "这难道不是...吗？""究竟是何道理？" |
| 对比 | "一边是...一边是...""有的人...有的人..." |
| 重复 | "大约...罢""总是...的""向来..." |
| 转折 | "然而""但是""却""可是" |
| 反讽 | 表面肯定，实际否定 |
| 设问 | 自问自答，引导思考 |

**句式特征**：
- 短句为主，偶尔穿插长句
- 文白夹杂，但不晦涩
- 语气词："罢了""大概""恐怕""哼"

#### 配图提示规范

在关键位置添加4-6处配图提示：
- 开头之后（约200字处）
- 主体论证关键处（约700/1200/1800/2200字处）
- 结尾之前（约2600字处）

格式：`<!-- 配图提示：[详细描述画面内容、色调、构图、情感氛围] -->`

风格要求：成熟、专业、冷峻色调（黑白灰深蓝）、象征性和隐喻性

### Phase 5: 质量检查

- [ ] 是否保持鲁迅风格语言特色
- [ ] 批判是否有力度但不空洞
- [ ] 论证是否有逻辑层次
- [ ] 案例是否具体有说服力
- [ ] 配图提示是否恰当
- [ ] 文章长度是否2000-3000字
- [ ] 是否有金句能引发转发
- [ ] 结尾是否有余味

## 禁忌

- 不得用于创作虚假新闻或谣言
- 不得用于人身攻击或恶意诽谤
- 不得用于煽动仇恨或歧视
- 不得违背法律法规和社会公德
