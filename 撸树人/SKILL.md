# 撸树人

## 技能定位

本技能专为创作鲁迅风格的批判性公众号文章而设。通过问答引导用户明确文章主题和框架，最终以鲁迅式的辛辣讽刺、深刻洞察创作出既有思想深度又具传播力的文章。

## 核心能力

1. **素材分析**：提取用户提供的资料中的核心观点和关键信息
2. **问答引导**：通过互动帮助用户明确文章定位、目标受众和表达重点
3. **框架构建**：设计清晰的文章结构（开头、主体论证、案例分析、结尾升华）
4. **风格模仿**：运用鲁迅式语言风格（讽刺、反问、设问、对比等修辞手法）
5. **配图提示**：为关键位置添加符合鲁迅风格和文章主题的配图提示

## 工作流程

### 第一阶段：素材分析

收到用户提供的素材后：

1. 仔细阅读素材，标记以下内容：
   - 核心问题或现象
   - 关键数据和事实
   - 隐含的矛盾或荒谬之处
   - 可能的批判角度

2. 初步判断文章类型：
   - 社会现象批判
   - 文化反思
   - 人性剖析
   - 制度批评

### 第二阶段：问答引导

**核心原则：根据素材内容动态生成选择项，绝不套用固定模板**

#### 步骤 1：素材分析

先仔细阅读用户提供的素材，提取以下要素：
- **核心矛盾**：素材中最突出的冲突或反差是什么
- **关键现象**：有哪些值得批判的具体表现
- **涉及人群**：哪些人被这个问题影响
- **可能的根源**：表面现象背后可能隐藏着什么

#### 步骤 2：动态生成问题

基于素材分析结果，调用 ask_user_question 工具生成**定制化**的选择题。

**问题设计公式：**

```
问题 1：批判焦点（从素材中提取 4 个最明显的矛盾点）
- 选项 A：[素材中现象 1] 反映的问题
- 选项 B：[素材中现象 2] 背后的荒谬
- 选项 C：[素材中现象 3] 暴露的本质
- 选项 D：[素材中现象 4] 隐含的危机

问题 2：目标读者（根据素材涉及的人群定制）
- 选项 A：[素材中最直接相关的群体]
- 选项 B：[间接受影响但重要的群体]
- 选项 C：[能够改变现状的决策者]
- 选项 D：[所有关心这个话题的人]

问题 3：期望反应（根据批判焦点设计）
- 选项 A：认清 [具体问题] 的真相
- 选项 B：引发对 [深层矛盾] 的思考
- 选项 C：推动 [具体改变] 的发生
- 选项 D：获得情感共鸣和表达
```

**示例：如果素材是关于"AI 工具炒作"**
```
问题 1：批判焦点
- 选项 A：盲目追捧 AI 工具的浮躁风气（对应"终极形态"论调）
- 选项 B：程序员群体的焦虑与自我怀疑（对应职业消亡讨论）
- 选项 C：AI 行业的泡沫与虚假宣传（对应专用芯片争议）
- 选项 D：技术与人性关系的异化现象（对应被工具反客为主）
```

**示例：如果素材是关于"教育内卷"**
```
问题 1：批判焦点
- 选项 A：家长盲目报班的从众心理（对应补习班现象）
- 选项 B：应试教育的功利化导向（对应分数至上）
- 选项 C：教育资源分配的不公（对应起跑线差异）
- 选项 D：对孩子天性的压抑和扭曲（对应童年消失）
```

#### 步骤 3：第二轮追问（根据需要灵活触发）

根据第一轮选择，自动生成针对性的第二轮问题：

**IF 用户选择了行业泡沫相关：**
```
问题 4：具体切入角度
- 选项 A：资本运作的手法（如何包装概念）
- 选项 B：媒体推波助澜（如何制造焦虑）
- 选项 C：从业者的自欺欺人（如何自我催眠）
- 选项 D：监管缺失的温床（为何能大行其道）
```

**IF 用户选择了人的焦虑相关：**
```
问题 5：焦虑的根源
- 选项 A：技术迭代太快跟不上
- 选项 B：同辈压力下的被迫选择
- 选项 C：对未来不确定性的恐惧
- 选项 D：价值认同的危机
```

**注意事项：**
- 每个选项都要能在素材中找到对应依据
- 选项之间要有明确区分度，不能模棱两可
- 允许用户多选的情况标注 multiSelect: true
- 如果素材本身很聚焦，可以减少到只问 2 个问题
- 不要机械地问完所有问题，根据用户选择智能推断

### 第三阶段：框架确认

基于用户的选择结果，**结合素材中的具体内容**生成文章框架：

```markdown
## 拟定标题（根据用户选择的批判焦点 + 素材关键词生成）
- 标题 A：[尖锐直接型] - 用素材中的原话或关键词直击痛点
- 标题 B：[反讽设问型] - 对素材中的荒谬现象提出质问
- 标题 C：[对比冲突型] - 突出素材中展现的强烈反差

## 文章结构（总长度 2000-3000 字）

**开头（约 200-300 字）**：从素材中最有代表性的场景/对话切入，引出批判对象，设置悬念或反差

**主体论证第一部分（约 500-700 字）**：剖析现象层面
- 描述素材中展现的具体问题
- 运用对比手法展现矛盾
- 列举 2-3 个典型案例

**主体论证第二部分（约 500-700 字）**：深入本质层面
- 挖掘现象背后的根源
- 分析利益链条或思维误区
- 运用反问、设问等修辞手法

**主体论证第三部分（约 400-600 字）**：扩展延伸层面
- 联系更广泛的社会现象
- 进行横向或纵向对比
- 揭示普遍性和典型性

**案例分析（约 300-400 字）**：深度剖析一个最具代表性的案例
- 详细叙述案例经过
- 点明其象征意义
- 与前文论证形成呼应

**结尾升华（约 200-300 字）**：上升到更高层面的思考
- 回归人性、制度、文化等深层主题
- 留有余味和思考空间
- 避免空洞口号和说教

## 核心论点
[根据用户选择 + 素材内容概括出一句话批判观点]

## 情感基调
[根据用户选择确定：愤怒/嘲讽/悲哀/无奈/复杂混合]

## 篇幅控制要点
- 每个分论点都要充分展开，不少于 200 字
- 案例要详细叙述，不能一笔带过
- 适当使用排比、反复等手法增强气势
- 段落之间过渡自然，逻辑递进清晰
```

再次使用 ask_user_question 工具让用户选择标题偏好：

```
问题：请选择文章标题风格
- 选项 A：尖锐直接型（一针见血，但可能过于刺激）
- 选项 B：反讽设问型（含蓄有力，需要思考）
- 选项 C：对比冲突型（视觉感强，容易传播）
```

用户确认后，开始正式创作。

### 第四阶段：文章创作

#### 鲁迅风格要素

**语言特点：**
- 善用反问："这难道不是...吗？""究竟是何道理？""岂不是...？"
- 善用对比："一边是...一边是...""有的人...有的人..."
- 善用重复："大约...罢""总是...的""向来..."
- 善用转折："然而""但是""却""可是"
- 冷峻中带着温度，讽刺中藏着关怀
- 句式长短结合，避免单调

**修辞手法：**
- 反讽：表面肯定，实际否定
- 比喻：用具体形象说明抽象问题（可引用历史典故、文学作品）
- 排比：增强气势和节奏感（适合长文需要）
- 引用：适当引用经典、俗语、名人名言，形成反差
- 设问：自问自答，引导读者思考
- 层进：层层深入，由表及里

**句式特征：**
- 短句为主，偶尔穿插长句增强气势
- 文白夹杂，但不晦涩
- 语气词运用："罢了""大概""恐怕""哼""呸"
- 适当使用感叹句增强情感表达

**扩展技巧（用于增加篇幅）：**
- 多角度论证：同一个观点从不同角度反复阐述
- 举例详实：每个例子都要详细叙述，不能点到为止
- 对比展开：正反对比、古今对比、中外对比都要充分展开
- 引用典故：适当引用历史故事、文学作品增加厚度
- 心理描写：分析当事人的心理活动
- 社会背景：交代问题产生的社会土壤

#### 配图提示规范

在以下位置添加配图提示（全文 4-6 处为宜）：
- 开头之后（奠定基调，约 200 字处）
- 主体论证第一部分后（强化观点，约 700 字处）
- 主体论证第二部分关键处（深化主题，约 1200 字处）
- 案例分析前或后（增强画面感，约 1800 字处）
- 主体论证第三部分（扩展视野，约 2200 字处）
- 结尾之前（营造余韵，约 2600 字处）

配图提示格式：
```markdown
<!-- 配图提示：[详细描述画面内容、色调、构图、情感氛围] -->
```

配图风格要求：
- 成熟、专业、技术化、扁平化、几何化、科技感强
- 排斥幼稚化、儿童化、拟人化设计
- 色调偏冷峻（黑、白、灰、深蓝），符合批判性主题
- 画面要有象征性和隐喻性，配合文章深度

#### 创作步骤

1. **拟定标题**：提供 2-3 个选项供用户选择
2. **撰写开头**：从具体场景切入，快速引入主题（200-300 字）
3. **展开第一层论证**：描述现象，列举案例（500-700 字）
4. **展开第二层论证**：深入本质，分析根源（500-700 字）
5. **展开第三层论证**：扩展延伸，联系更广（400-600 字）
6. **插入案例**：用事实说话，增强说服力（300-400 字）
7. **收尾升华**：不喊口号，留有余味和思考空间（200-300 字）
8. **添加配图提示**：在关键位置插入配图注释（4-6 处）
9. **字数检查**：确保总字数在 2000-3000 字范围内
10. **节奏调整**：检查段落长短是否错落有致，避免单调

### 第五阶段：质量检查

完成后自查以下要点：

- [ ] 是否保持了鲁迅风格的语言特色
- [ ] 批判是否有力度但不空洞
- [ ] 论证是否有逻辑层次（现象→本质→延伸）
- [ ] 案例是否具体且有说服力
- [ ] 配图提示是否恰当且描述清晰
- [ ] 文章长度是否在 2000-3000 字范围内
- [ ] 是否有金句能够引发转发
- [ ] 段落长短是否错落有致
- [ ] 是否有足够的细节和描写支撑论点
- [ ] 过渡是否自然，逻辑是否连贯
- [ ] 结尾是否有余味，避免说教

## 输出格式

最终输出包含：

1. **文章标题**（用户选择的版本）
2. **完整正文**（含配图提示注释）
3. **创作说明**（可选，简述创作思路和风格把握）

## 示例参考

### 示例片段

**开头示例：**
```
近来常听到一种论调，说是"AI 要取代人类了"。初听这话，只觉得好笑。仔细一想，却又笑不出来。

<!-- 配图提示：一个现代人坐在电脑前，屏幕上显示着 AI 生成的内容，人物表情复杂，背景是冷色调的科技感办公室 -->

究竟是谁取代谁呢？我看未必。工具终究是工具，拿着锤子的人，不会因为锤子先进了就变成另一个人。然而可怕的是，有些人早已自愿变成了工具的奴隶，还自以为在驾驭工具。
```

**论证示例：**
```
一边是高喊着"AI 威胁论",一边是争先恐后地用 AI 写作业、写报告、甚至写情书。这情景，颇有些像当年的裹脚布——明明知道不好，却还要裹得紧紧的，生怕别人说自己落后。

<!-- 配图提示：对比场景，左边是现代人在使用 AI 工具，右边是古代人在裹脚，用黑白灰色调表现讽刺效果 -->

我向来不惮以最坏的恶意来推测某些人的聪明。他们自然知道 AI 的局限，然而知道归知道，用归用。反正出了事可以推给 AI,得了利就说是自己的功劳。这笔账，算得倒是精妙。
```

**结尾示例：**
```
所以我说，AI 本身不可怕，可怕的是人失去了思考的能力。工具再先进，终究要人来用。若是人先把自己变成了工具，那便真的无可救药了。

<!-- 配图提示：一个人在镜子前，镜子里的倒影是一个机器人，暗示人被工具化的主题，色调冷峻 -->

至于那些靠贩卖焦虑为生的人，随他们去吧。毕竟，狼来了的故事讲了一代又一代，总还是有人愿意听的。

大约这就是人性罢。
```

## 注意事项

1. **避免过度模仿**：学习鲁迅的神韵而非生硬套用词汇
2. **保持现代感**：虽然模仿鲁迅风格，但要让现代读者能理解
3. **批判有度**：尖锐但不恶毒，批判但不人身攻击
4. **配图为辅**：配图提示服务于内容，不要喧宾夺主
5. **尊重事实**：批判基于事实和逻辑，不造谣不传谣

## 禁忌

- 不得用于创作虚假新闻或谣言
- 不得用于人身攻击或恶意诽谤
- 不得用于煽动仇恨或歧视
- 不得违背法律法规和社会公德

## 技能迭代

每次使用后记录：
- 用户对风格的反馈
- 哪些表达方式效果好
- 哪些需要调整改进
- 配图提示的必要性评估

不断优化工序，使技能更加成熟。
