# 用户画像模块

> 分析用户行为，推断用户身份和擅长领域

## 核心功能

1. **行为统计**：记录用户各类行为次数和主题
2. **身份推断**：根据行为分布推断用户主身份
3. **画像查询**：展示用户画像报告

## 行为类型

| action 类型 | 说明 | 触发场景 |
|-------------|------|---------|
| `ask_concept` | 询问概念 | 用户问"什么是..." |
| `ask_method` | 询问方法 | 用户问"怎么做..." |
| `write_code` | 写代码 | 用户让 AI 写代码 |
| `explain_code` | 解释代码 | 用户让 AI 解释代码 |
| `搬题` | 搬题行为 | 用户使用搬题姬 |
| `算法讲解` | 算法讲解 | AI 讲解算法给用户 |

## 身份推断规则

```
行为分布分析：
- 如果 ask_concept 次数最多，主题多为算法 → 身份可能是"算法学习者"
- 如果 搬题 次数多 → 身份可能是"OJ管理员/题目管理员"
- 如果 算法讲解 次数多 → 身份可能是"信奥教练/老师"
- 如果 write_code 次数多，主题多为项目 → 身份可能是"开发者"

置信度计算：
- 行为数据越多，置信度越高
- 行为分布越集中，置信度越高
```

## 执行流程

### 行为记录（自动）

每次触发墨池时，自动更新行为统计：

```
1. 判断本次行为类型（ask_concept/write_code/...）
2. read_file user_profile.txt
3. 找到对应的 action 块
4. edit_file 更新：
   - count: +1
   - topics: append 新主题（如果不在列表中）
5. 重新计算身份（如果行为分布变化明显）
```

### `/墨池 画像`

**执行流程**：

```
1. read_file user_profile.txt
2. AI 分析行为分布，生成报告
3. 输出给用户
```

**报告格式**：

```markdown
# 用户画像报告

## 推断身份

**主身份**：信奥竞赛教练
**次身份**：算法学习者、C++开发者
**置信度**：85%

## 行为统计

| 行为类型 | 次数 | 主要主题 |
|---------|------|---------|
| 询问概念 | 15 | 前缀和、差分、树状数组 |
| 搬题 | 8 | ABC450A、ABC450B |
| 算法讲解 | 5 | 快速排序、二分查找 |

## 建议复习

根据行为分析，建议复习：
- 差分（已学习 3 次，掌握程度：熟练）
- 树状数组（已学习 2 次，掌握程度：理解）
```
