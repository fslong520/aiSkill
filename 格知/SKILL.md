# 格知 — 一格一知识

## 技能说明

一键生成单张宫格排版的科普长图，支持**参考图约束**保持 IP 形象一致。用户提供科普主题和主 IP 形象（或由 AI 生成），AI 自动规划内容分区，将所有知识点浓缩在一张宫格图中输出。支持自定义宫格数量（4 宫格/6 宫格/9 宫格等），风格统一，适合小红书、公众号等平台直接发布。

## 核心差异

| 对比项 | 9 图科普（原版） | 宫格科普图（本技能） |
| --- | --- | --- |
| 输出数量 | 9 张独立图片 | **1 张**宫格合成图 |
| 排版方式 | 每张独立构图 | 单图内分区宫格排版 |
| IP 形象 | 需要参考图约束 | ✅ 同样支持参考图约束 |
| 宫格数量 | 固定 9 张 | 用户自选（4/6/9 等） |
| 使用场景 | 小红书轮播图 | 单图科普、知识卡片、朋友圈分享 |

## 执行流程

### Step 1：确认或生成主 IP 形象

询问用户："请先提供你的主 IP 形象图片（正面全身照），我将严格固定这个形象生成宫格科普图。"

**如果用户有参考图：**
- 请用户提供图片或图片 URL
- 记录图片 URL，后续生成必须使用此图作为 `source_image_urls` 约束

**如果用户没有参考图：**
- 询问用户想要的形象特征（动物/人物/颜色等）
- 使用 `imageGenerate` 生成标准形象
- 提示词模板："极简手绘涂鸦风格，【角色描述】，圆润可爱，纯白色身体，【特征颜色】点缀，极简线条，纯白背景，正面全身照"
- 记录生成的图片 URL 作为约束基准

**重要：必须获得标准形象图后，才能进入下一步！**

---

### Step 2：确认科普主题与宫格数量

询问用户两个关键信息：

**问题 1：** "你想科普什么主题？"
- 记录用户的科普主题
- 如果用户在触发技能时已经提供了主题，直接记录，不必重复询问

**问题 2：** "你想要几宫格的排版？"
- 🔲 **4 宫格**（2×2）：适合简洁对比、四个核心概念
- 🔲 **6 宫格**（2×3 或 3×2）：适合流程梳理、中等信息量
- 🔲 **9 宫格**（3×3）：适合完整知识体系、全面科普

**必须同时获得主题和宫格数后，才能进入下一步。**

---

### Step 3：规划宫格内容结构

**4 宫格内容框架（2×2）：**
1. 标题定义（What）——一句话说清楚是什么
2. 核心原理（How）——怎么运作的
3. 应用场景（Where）——用在哪里
4. 金句总结（Summary）——一句话记住它

**6 宫格内容框架（2×3）：**
1. 标题封面（Hook）——主题+吸引眼球的标语
2. 核心定义（What）——一句话说清楚是什么
3. 运作原理（How）——核心机制拆解
4. 核心价值（Why）——为什么重要/有什么好处
5. 应用场景（Where）——实际用在哪里
6. 金句总结（Summary）——一句话记住它

**9 宫格内容框架（3×3）：**
1. 标题封面（Hook）——主题+吸引眼球的标语
2. 痛点引入（Pain）——没有它会怎样
3. 核心定义（What）——一句话说清楚是什么
4. 内部拆解（Structure）——组成部分/步骤分解
5. 核心价值（Why）——为什么重要
6. 应用场景（Where）——实际用在哪里
7. 执行逻辑（How）——怎么一步步运作
8. 易混淆对比（VS）——和相似概念的区别
9. 金句总结（Summary）——一句话记住它

将规划好的每格内容简要告知用户，确认无误后进入生成环节。

---

### Step 4：生成单张宫格科普图

使用 `imageGenerate` 工具生成宫格图。**必须使用** `source_image_urls` **参数传入 Step 1 获得的主 IP 形象图。**

**Prompt 构建模板：**

```
严格参照参考图中的角色形象，生成一张{N}宫格科普知识长图，主题是"{用户主题}"。

【角色形象要求 - 必须完全一致】
{根据参考图描述形象特征}
角色必须出现在多个格子中，保持形象完全一致，不能变形！

【整体布局】
- {M}×{K}宫格排列
- 每个格子之间有清晰的分隔线
- 整体是一张完整的图片

【各格内容】
第1格：{该格标题} — {该格核心文案}
...（依次列出所有格子）

【视觉风格】
- 极简手绘涂鸦风格
- 黑白线条为主，可用1-2种强调色点缀
- 每格配有角色的不同动作/表情
- 文字中文，简练有力
- 整体90%留白

【文字要求】
- 所有文字必须是中文
- 每格标题加粗醒目
- 正文控制在15字以内
```

**生成参数：**
- model：`gemini-3.1-flash-image-preview`
- source_image_urls：**必须传入 Step 1 获得的主 IP 形象图 URL**
- aspect_ratio：根据宫格类型选择
  - 4 宫格 → `1:1`
  - 6 宫格 → `3:4`
  - 9 宫格 → `1:1`
- quality：`medium`

---

### Step 5：整理输出文档

使用 `create_file` 工具创建输出文档，包含：

1. **主 IP 形象图**（嵌入参考图）
2. **生成的宫格科普图**（嵌入图片）
3. **各格文案整理**
4. **发布建议**：推荐平台、标题、标签、正文模板
5. **主 IP 形象说明**

文档标题格式：`「{主题}」{N}宫格科普图`
