# 安全保护机制

文件整理的安全保障体系。

---

## 核心安全原则

| 原则 | 说明 |
|------|------|
| **不删除** | 只移动，不删除任何文件 |
| **先备份** | 操作前完整备份 |
| **可追溯** | 每一步都有日志 |
| **可恢复** | 一键撤销功能 |

---

## 备份机制

### 备份策略

```
移动-复制备份策略：
1. 先用mv移动原始文件到备份目录
2. 再用cp复制一份到目标位置
3. 原始文件保留在备份目录
```

### 备份目录结构

```
.neuralshelf_backup/
├── 2026-03-30_10-00-00/    # 时间戳快照
│   ├── original_files/     # 原始文件备份
│   ├── operation_log.json  # 操作日志
│   └── snapshot_info.json  # 快照信息
├── 2026-03-30_11-30-00/
│   └── ...
```

## 项目结构保护

### 项目识别标记

| 标记 | 说明 |
|------|------|
| `.git/` | Git仓库 |
| `.idea/` | IDE配置 |
| `package.json` | Node.js项目 |
| `pom.xml` | Java项目 |
| `.venv/` | Python虚拟环境 |

### 保护策略

- 检测到项目标记 → 项目目录整体保留
- 不打散项目结构
- 项目相关文件集中管理

## 日志系统

### 日志内容

```json
{
  "timestamp": "2026-03-30T10:00:00",
  "operation": "move",
  "source": "/home/user/Downloads/file.pdf",
  "target": "/home/user/Documents/Papers/file.pdf",
  "backup": "/home/user/.neuralshelf_backup/2026-03-30_10-00-00/original_files/file.pdf"
}
```

### 日志用途

- 追溯每一步操作
- 支持一键撤销
- 支持恢复到任意时间点

---

## 恢复机制

### 一键撤销

```bash
# 撤销最近一次整理
neuralshelf undo

# 执行过程：
# 1. 读取最近快照的操作日志
# 2. 按日志逆向操作
# 3. 恢复原始文件位置
# 4. 清理目标位置的文件
```

### 时间点恢复

```bash
# 恢复到指定时间点的状态
neuralshelf restore --time "2026-03-30 10:00"
```

## 智能去重机制

### MD5哈希检测

```python
import hashlib

def get_file_md5(filepath):
    with open(filepath, 'rb') as f:
        return hashlib.md5(f.read()).hexdigest()
```

### 去重策略

| 情况 | 处理方式 |
|------|---------|
| MD5完全相同 | 标记为重复，建议删除副本 |
| 名称相同但内容不同 | 保留两个，添加区分标记 |
| 内容相似但名称不同 | 提示用户判断 |

### 去重报告

```markdown
## 重复文件报告

原始文件数：100
去重后文件数：85
节省空间：2.3GB

重复文件列表：
- file1.pdf = file2.pdf（MD5相同）
- photo1.jpg ≈ photo2.jpg（内容相似）
```
