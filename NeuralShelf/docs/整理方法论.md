# 四阶段整理方法论

NeuralShelf 采用神经网络驱动的四阶段整理流程。

---

## 阶段一：神经感知分析阶段

**目标**：分析文件内容、类型、关联关系

### 分析维度

| 维度 | 分析内容 | 输出结果 |
|------|----------|----------|
| **内容分析** | 文件名、关键词、内容特征 | 内容主题判断 |
| **类型分析** | 扩展名、文件结构 | 文件类型归类 |
| **关联分析** | 文件间的命名关联、时间关联 | 文件关系图 |
| **项目识别** | .git、.idea、package.json等标记 | 项目边界确定 |

### 输出内容

```
文件列表 → 内容特征 → 类型归类 → 关联关系 → 项目边界
```

---

## 阶段二：神经决策分类阶段

**目标**：基于分析结果，智能决定整理方案

### 分类决策原则

| 原则 | 说明 |
|------|------|
| **内容导向** | 按内容相关性分组，不是按文件类型 |
| **项目保护** | 项目文件整体保留，不打散 |
| **去重优先** | MD5哈希检测，识别完全相同的文件 |
| **命名智能** | 分析命名规律，批量命名优化 |

### 分类输出格式

```markdown
## 整理方案

### 类别一：[主题名称]
- 文件列表：[相关文件]
- 目标位置：[目标文件夹]
- 处理方式：移动/复制

### 类别二：[主题名称]
[同上结构]

### 重复文件（建议删除）
- 文件列表：[重复文件]
- 原文件保留：[保留哪个]
```

---

## 阶段三：神经执行整理阶段

**目标**：安全执行整理方案

### 执行流程

```
1. 创建备份目录 → 备份所有原始文件
2. 创建目标结构 → 按方案建立文件夹
3. 执行文件操作 → mv或cp文件到目标位置
4. 生成操作日志 → 记录每一步操作
```

### 安全原则

- **先备份后操作**：任何操作前先完整备份
- **移动优于删除**：不删除任何文件，只移动
- **日志完整记录**：每一步都有日志可追溯

---

## 阶段四：神经恢复保障阶段

**目标**：提供一键撤销恢复功能

### 恢复机制

```bash
# 查看整理历史
neuralshelf history

# 一键撤销最近一次整理
neuralshelf undo

# 恢复到指定时间点
neuralshelf restore --time "2026-03-30 10:00"
```

### 恢复原理

- 每次整理前创建备份快照
- 操作日志记录每一步
- 恢复时按日志逆向操作

---

## 整理成功判定标准

| 标准 | 要求 |
|------|------|
| **完整性** | 所有文件都有目标位置 |
| **可追溯** | 操作日志完整记录 |
| **可恢复** | 备份快照完整保存 |
| **结构清晰** | 目标文件夹层次合理 |
