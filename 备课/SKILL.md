# 信息学奥赛准备

## 概述

准备CSP-J/S、NOI、GESP和电子学会认证的讲义材料。为K-12学生提供标准化模板、课程指导，用于教授C++编程、数据结构和算法。

内容对齐NOI大纲的难度级别（入门级/CSP-J，提高级/CSP-S，NOI级），同时支持GESP和电子学会认证系统。

**自动去AI味**：所有生成的内容会自动应用去AI味原则，确保文字自然流畅，避免AI写作的常见痕迹（过度强调、机械结构、冗余表达等）。

## 迭代优化机制

技能支持自我迭代和改进。根据用户使用情况和反馈优化文件，让技能越来越好用。

### 优化触发条件

这些情况会触发优化：

**用户主动触发**
- 用户给出反馈（正面或负面）
- 用户发现错误或不清晰的内容
- 用户提出新需求或建议
- 用户频繁使用某些功能而忽略其他功能
- 用户主动提出改进建议

**自我优化**
每次使用后收集信息：
- 用户对生成内容的修改
- 用户评价
- 用户是否直接采用还是大幅改写

每周根据收集的信息更新文件，让技能更贴合实际需求。

### 优化方向

可以优化的方面：

- **内容完整性**：补充缺失的知识点或案例
- **清晰度**：优化模糊表述，增加类比和例子
- **实用性**：根据使用情况调整模板和格式
- **教学效果**：根据反馈调整教学方法和语言风格
- **技术准确性**：修正技术错误或过时内容
- **可操作性**：让指导更具体、易执行

### 优化执行流程

需要优化时：

1. **记录反馈**：用`update_memory`工具记录反馈和使用情况
2. **分析问题**：分析问题根源和影响范围
3. **制定方案**：提出具体优化方案
4. **执行优化**：用`create_rule`工具更新技能文件
5. **验证效果**：下次使用时验证优化效果

### 优化记录

记录每次优化：
- 优化的原因和背景
- 修改的具体内容
- 期望达到的效果
- 实际效果（后续追踪）

形成技能演进历史，便于回溯和改进。

### 持续改进循环

```
使用技能 → 收集反馈 → 分析问题 → 优化技能 → 验证效果 → 继续使用
    ↑                                                    ↓
    ←─────────────────── 持续改进循环 ←──────────────────┘
```

不断重复这个循环，让技能在使用中完善，逐渐适应具体需求和教学风格。

## 核心任务类别

### 任务1：确定课程内容

参考`references/备课规则.md`获取完整课程结构。

**入门级 (CSP-J，难度系数1-5)**
- 基础：计算机概念、操作系统、编程环境
- C++基础：数据类型、输入输出、运算符、控制结构、数组、字符串、函数、递归
- 数据结构：数组、链表、栈、队列、基础树、基础图
- 算法：枚举、模拟、贪心、递归、二分查找、基础排序、DFS/BFS、简单DP、高精度
- 数学：算术、数制、初等数学、初等数论、离散数学

**提高级 (CSP-S，难度系数5-8)**
- 高级C++：类和对象、STL、运算符重载
- 数据结构：高级线性结构、并查集、特殊树（堆、树状数组、线段树、字典树、平衡树）、图结构、哈希表
- 算法：复杂度分析、分治、高级排序、KMP、搜索优化、图算法（最小生成树、最短路径、拓扑排序）、高级DP
- 数学：高等数学、数论、组合数学、线性代数

**NOI级 (难度系数7-10)**
- 高级编程：面向对象设计、复杂数据结构（树链剖分、可持久化结构）
- 高级算法：分块、离线处理、字符串算法（后缀数组、后缀自动机）、高级图算法（网络流、2-SAT）、DP优化

### 任务2：创建讲义材料

### 任务4：遵循教学方法

**教学理念（来自`references/备课规则.md`第1.3节）：**
- 支架式学习（从易到难、循序渐进）
- 理论实践结合
- 强调算法思维和问题解决
- 重视代码实践和调试能力
- **关键**：不要提供完整代码，只提供关键步骤、关键代码片段或留空关键部分并添加注释
- 学生自己完成和调试代码

**教学方法（第3.3节）：**
- 使用启发式、探究式教学
- 专注于算法思维，不仅仅是语法
- 鼓励一题多解，培养创造力
- 强调代码规范和良好编程习惯

**目标受众：14岁中学生**

调整内容匹配14岁学生的认知发展和兴趣。

**语言风格要求：**
- **机智幽默**：用引人入胜、轻松的语言保持兴趣
- **适合年龄**：避免幼稚内容（不要小看他们），但避免过于学术化枯燥
- **贴近生活**：将概念与青少年的经历和兴趣（游戏、社交媒体、日常生活）联系起来
- **清晰但不简化**：适当使用精确术语，但以可理解的方式解释
- **对话式语调**：像与聪明青少年交谈一样写作，而不是小孩子
- **自然流畅**：去除AI痕迹，避免机械表达，让文字有人味

**经常使用类比（多用打比方）：**

用生动的类比使抽象的编程概念具体化：
- **变量**：像贴标签的盒子存储物品
- **函数**：像有输入和输出的专用工具或机器
- **循环**：像游戏中或锻炼中的重复动作
- **数组**：像编号的储物柜行或剧院的座位
- **指针**：像信封上的地址，告诉你在哪里找到真正的信件
- **递归**：像俄罗斯套娃或调用自身解决更小版本问题的函数
- **栈/队列**：像盘子堆（后进先出）或售票处的等待队列（先进先出）
- **二分查找**：像通过反复将范围减半来猜1-100之间的数字
- **贪心算法**：像在自助餐上总是拿最大的那片披萨（局部最优选择）
- **动态规划**：像记住数学问题的答案，这样就不用重新计算

**参与策略：**
- 从贴近生活的场景或有趣的问题开始
- 在有帮助时使用游戏参考和比较
- 包含关于现实世界应用的"你知道吗？"时刻
- 添加与编程相关的轻松笑话或双关语
- 创建青少年关心的问题场景（排名系统、游戏机制、社交功能）
- 庆祝进步，鼓励在困难概念上的坚持

**平衡复杂性：**
- 将复杂主题分解为可管理的部分
- 即使没有实际图像也使用视觉描述
- 先解释"为什么"再解释"如何"，先建立直觉
- 适当挑战，14岁青少年在适当激励下可以处理抽象概念
- 展示算法能力的"酷"因素（游戏AI、推荐系统等）

**代码风格要求：**
- 所有左大括号`{`在单独一行，不能与任何代码在同一行
- 适用于：函数定义、类定义、结构体定义、if/else语句、for/while循环、switch语句和所有使用大括号的地方

**去AI味核心原则（自动应用）：**

生成讲义时自动遵循以下原则，确保内容自然有人味：

1. **删除填充短语**
   - 避免"可以"、"能够"、"能够帮助"等冗余表达
   - 直接说"这样做"而不是"能够这样做"
   - 删除"在这个时间点"、"在……的情况下"等虚词

2. **打破公式结构**
   - 避免"不仅仅……而且……"的二元对比
   - 不要用三段式列举（三项），改用两项或四项
   - 避免戏剧性的分段和修辞性设置

3. **变化节奏**
   - 混合使用短句和长句
   - 不要让连续三个句子长度相同
   - 段落结尾方式要多样化

4. **信任读者**
   - 直接陈述事实，不要过度解释
   - 跳过软化语言和手把手引导
   - 相信学生能理解，不要重复解释

5. **删除AI高频词**
   - 避免"此外"、"确保"、"强调"、"关键"、"体现"等词
   - 不用"标志着"、"彰显了"、"凸显"等夸张表达
   - 避免"培养"、"促进"、"提升"等模糊动词

6. **去除机械强调**
   - 减少粗体使用，只强调真正重要的内容
   - 不要用"必须"、"应该"、"务必"等命令式语气
   - 避免括号中的重复解释

7. **增加个性**
   - 适当使用"我"等第一人称表达观点
   - 可以有真实的反应和感受
   - 允许一些不确定性："我倾向于认为……"
   - 添加轻松的幽默和双关语

8. **具体而非模糊**
   - 用具体例子代替模糊主张
   - "这个算法很快" → "这个算法能在0.01秒内处理10万个数据"
   - "很多人喜欢" → "在2024年的调查中，85%的学生表示喜欢"

9. **避免空洞结论**
   - 不要用"激动人心的时代即将到来"等套话
   - 用具体行动代替："下次课我们会实现……"
   - 删除"这代表了重要的一步"等无意义表述

10. **自然表达**
    - 使用"是/有"等简单动词，不要复杂化
    - 避免过度使用分词短语"-ing"
    - 不要用破折号来增加戏剧性效果
    - 表情符号适度使用，不要过度装饰

## 去AI味示例对比

展示讲义内容如何通过去AI味变得更自然：

**AI风格（修改前）：**
> 在这个部分，我们将会深入探讨排序算法。排序是计算机科学中至关重要的主题，它强调了数据组织的重要性。此外，排序算法在实际应用中发挥着关键作用。为了更好地理解，我们将学习冒泡排序和选择排序。

**去AI味后（修改后）：**
> 这节课学排序。排序就是把数据按顺序排列——从大到小或从小到大。我们会学两种方法：冒泡排序（像冒泡一样把大的往上顶）和选择排序（每次选最小的）。

**主要改进：**
- 删除了"将会深入探讨"、"至关重要的"等AI高频词
