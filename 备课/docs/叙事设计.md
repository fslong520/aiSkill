# 课程叙事设计

课程设计的核心问题不是教学问题，是叙事问题。

电影工业花了近百年解决一个问题：怎么让人类注意力在固定时长内持续集中。

---

## 一、情绪曲线：棘轮效应

**规则**：情绪走势是波浪形的（升→小降→升更高→小降→升到最高），每次"小降"的谷底都比上一个谷底更高。

**关键原则**：张力是不可转让的资产。

### 翻车案例

- A老师花了1小时47分钟把情绪从2分拉到8.5分
- B老师上场第一句话："大家好，我先简单介绍一下自己"
- 情绪直接坠回3分，B老师得从零开始重建信任

### 备课应用

- ✅ 每个知识点的引入要比上一个更有冲击力
- ✅ 练习/互动的"小降"后，下一个知识点要升得更高
- ❌ 不要在高潮后突然插入无关内容
- ❌ 不要在课程中途换主讲风格

### 情绪曲线示例（2小时课程）

```
导入（游戏场景）→ 2分
  ↓ 概念讲解
基础语法 → 5分
  ↓ 小练习（小降）→ 4分
  ↓ 进阶用法
核心技巧 → 7分
  ↓ 挑战题（小降）→ 6分
  ↓ 实战案例
综合应用 → 9分（最高点）
```

## 二、信息调度：植入-揭示分离

**规则**：前半段密集植入信息，后半段揭示含义，中间留出"认知痒"的空白期。

**什么是"认知痒"**：
- 观众隐约觉得哪里不对，但就是拼不上
- 这段空白让你坐立不安，但又舍不得走
- 在等待答案的过程中，注意力高度集中

### 翻车案例

- 讲完一个案例，立刻揭示底层逻辑
- 信息的植入和揭示几乎同时完成——即种即收，杀死悬念

### 备课应用

- ✅ 先抛出问题/悬念，让子弹飞一会儿
- ✅ 中间穿插其他内容（但要有关联）
- ✅ 最后揭示答案，形成"原来如此"的顿悟感
- ❌ 不要讲完问题立刻给答案

### 植入-揭示示例

```
【植入】开场：为什么这道题N=1000能过，N=10000就超时了？
    ↓
【空白期】讲解时间复杂度概念、O(n²)和O(n log n)的区别
    ↓ （学生心里一直惦记着那道题）
【揭示】回到原题：因为你的算法是O(n²)，改成O(n log n)就能过
```

## 三、赌注阶梯：只升不降

**规则**：每进一层，赌注就跳一个台阶。台阶之间是离散的跳跃，只升不降。

### 翻车案例

- "这个知识点考试必考！"→"其实也不一定……"
- 赌注从楼梯变成了弹簧，上去又下来
- 学生很快就学会："老师说很重要，其实也就那样"

### 备课应用

- ✅ 知识点的重要性要递进：考试会考→竞赛必考→省选难度
- ✅ 例题难度要递进：基础→提高→挑战
- ❌ 不要反复强调"很重要"然后又打折扣

### 赌注阶梯示例

```
基础题：掌握基本语法，考试能拿分
    ↑ 阶梯
进阶题：理解算法思想，竞赛能过初赛
    ↑ 阶梯
挑战题：综合运用能力，冲刺省一
```

## 四、课程设计三阶段

| 阶段 | 类比 | 内容 |
|------|------|------|
| 结构设计 | 建筑设计图 | 目标、模块划分、情绪曲线、赌注阶梯 |
| 内容编排 | 室内设计 | 教学策略、植入-揭示设计、练习安排 |
| 内容生产 | 施工装修 | 话术脚本、代码示例、类比比喻 |

## 五、自检清单

**Q1：情绪曲线有没有回到过起点？**
- 如果有→那个位置就是学生流失的位置

**Q2：植入信息后有没有留足够空白再揭示？**
- 如果没有→内容会"正确但无聊"

**Q3：赌注是只升不降的楼梯，还是上下弹跳的弹簧？**
- 如果是弹簧→每次"其实也不难"都在消解信任

## 六、叙事技巧速查

| 技巧 | 电影案例 | 课程应用 |
|------|---------|---------|
| 悬念开场 | 《盗梦空间》开场直接是高潮片段 | 用一道难题/反直觉现象开场 |
| 认知痒 | 陀螺的意义直到结尾才揭示 | 先抛问题，中间讲知识，最后揭示 |
| 情绪锚点 | Cobb对Mal的执念贯穿全片 | 用一个核心案例贯穿整节课 |
| 赌注升级 | 每层梦境风险递增 | 例题难度递进，重要性递增 |
| 收束呼应 | 开头的陀螺结尾再次出现 | 结尾回到开场的难题，展示掌握后的解法 |
