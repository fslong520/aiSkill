# ADDIE 阶段 1：Analysis（分析）

> **定位**：搞清楚"教谁、教什么"
> **对应 ADDIE**：Analysis 阶段
> **前置**：无（备课起点）
> **后续**：→ [02-Design 设计](./02-Design设计.md)

---

## 阶段目标

明确教学起点和终点，为后续设计提供依据。

---

## 四大分析维度

### 1. 学习者分析

| 关键问题 | 说明 |
|---------|------|
| 学生多大？ | 目标受众：14 岁左右中学生 |
| 基础如何？ | 零基础/有 C 基础/有 Python 基础 |
| 学过什么？ | 前置知识点清单 |
| 认知特点是什么？ | 抽象思维发展阶段，需要具体类比支撑 |

**输出**：学习者画像

---

### 2. 需求分析

| 关键问题 | 说明 |
|---------|------|
| 这节课要解决什么问题？ | 学生的知识缺口 |
| 考试考什么？ | CSP-J/S、GESP、电子学会考纲 |
| 竞赛需要什么？ | 实际竞赛能力要求 |

**输出**：学习目标清单（用行为动词描述：能实现/能解决/能分析）

---

### 3. 内容分析

| 关键问题 | 说明 |
|---------|------|
| 知识点之间的逻辑关系？ | 先学什么、后学什么 |
| 前置知识是什么？ | 必须先掌握的知识点 |
| 难点在哪？ | 学生容易卡住的地方 |

**输出**：知识结构图

---

### 4. 环境分析

| 关键问题 | 说明 |
|---------|------|
| 课时多长？ | 2 小时/3 小时/半天 |
| 机房还是教室？ | 有没有电脑实操条件 |
| 有没有 OJ 系统？ | 智国学堂 OJ：https://fslong.iok.la/ |

**输出**：教学约束条件

---

## 级别对照表

| 级别 | 难度系数 | 目标认证 | 核心内容 |
|------|---------|---------|---------|
| 入门级 | 1-5 | CSP-J、GESP 1-4 级 | C++ 基础、枚举、模拟、贪心、DFS/BFS、简单 DP |
| 提高级 | 5-8 | CSP-S、GESP 5-6 级 | STL、分治、图算法、高级 DP、数论 |
| NOI 级 | 7-10 | NOI、省选 | 复杂数据结构、字符串算法、网络流、DP 优化 |

详细内容参见 [references/备课规则.md](../references/备课规则.md)

---

## 分析阶段检查清单

- [ ] 确认学生级别（入门级/CSP-J，提高级/CSP-S，NOI 级）
- [ ] 分析前置知识（学生学过什么？还缺什么？）
- [ ] 明确教学目标（用行为动词描述：能实现/能解决/能分析）
- [ ] 识别难点和易错点
- [ ] 确认教学约束（课时、机房/OJ、设备）

---

## 输出物

1. **学习者画像**：一句话描述学生现状
2. **教学目标清单**：3-5 条，用行为动词描述
3. **知识结构图**：知识点先后关系
4. **教学约束条件**：课时、环境、设备

完成后进入 → [02-Design 设计](./02-Design设计.md)
