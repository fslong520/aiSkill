## Step 1：创建讲义/PPT

使用模板 [assets/讲义ppt模板.md](../assets/讲义ppt模板.md)

### 问题驱动式讲义结构

每个算法知识点遵循以下结构（**禁止记忆模板式**）：

```
问题驱动式讲义章节结构：
1. 导入：抛出具体问题场景（学生能感知的问题）
2. 引导：拆解为 2-4 个具象问题，逐个引导学生发现答案
3. 骨架：给注释框架，学生自己填核心代码
4. 揭示：展示完整模板 = 推导的结论
5. 巩固：练习1（纸上手动模拟）
6. 例题：OJ 提交（老师带做）
7. 变式：OJ 提交（独立完成）
8. 练习：更多 OJ 独立练习
```

## Step 3：编写代码示例

### 代码规范（竞赛风）

- 左大括号 `{` 独居一行
- 变量名短（不逾五字符）
- 优先用全局变量
- **不付全码**，只留关键步或填空

### 问题驱动式代码组织（新增）

代码展示也遵循"引导发现"逻辑：

| 做法     | ❌ 错误（记忆模板式）     | ✅ 正确（问题驱动式）                                     |
| -------- | ------------------------ | -------------------------------------------------------- |
| 展示方式 | 直接贴完整模板，逐行解释 | 先贴注释骨架，学生逐行填，最后展示完整代码               |
| 顺序     | 模板 → 例子 → 练习       | 问题 → 拆解 → 骨架 → 填充 → 模板（=结论）                |
| 代码注释 | 只有说明性注释           | 留有引导性问题注释（`// 往哪走？`、`// 什么情况停下？`） |

### 坐标语义规范（二维数组/地图题）

```cpp
// ✅ 正确：y 是第一维（行），x 是第二维（列）
void dfs(int y, int x) {
    if (y < 0 || y >= n || x < 0 || x >= m) return; // 越界检查
    if (vis[y][x] || a[y][x] == 0) return;           // 已访问/不可走
    vis[y][x] = 1;
    for (int i = 0; i < 4; i++)
        dfs(y + dx[i], x + dy[i]);
}
// 注意：dx 改变 y（行），dy 改变 x（列）
```

### 代码质量要求

- 详注（每关键行有注）
- 时空复杂度析
- 易错点提醒
- 变量名合竞赛规

## Step 4：设计分层作业

| 层级 | 题量 | 难度             | 目标         |
| ---- | ---- | ---------------- | ------------ |
| 基础 | 3 道 | 课堂知识直接应用 | 巩固基本概念 |
| 提高 | 3 道 | 需要综合运用     | 培养解题思维 |
| 挑战 | 2 道 | 超出课堂范围     | 冲刺竞赛     |

**注意**：作业题不与课堂例题重复。

### 讲练结合递进练习序列（新增）

每个知识点模块的练习序列必须从最基础的练习开始：

| 序号       | 形式                | 目标                   | 示例                       |
| ---------- | ------------------- | ---------------------- | -------------------------- |
| **练习1**  | 纸上手动模拟        | 巩固核心逻辑，不可跳过 | 在 3×3 网格上模拟 DFS 过程 |
| **例题**   | OJ 提交（老师带做） | 演示完整解题流程       | 数一数有几块陆地           |
| **变式**   | OJ 提交（独立完成） | 对例题的简单改动       | 求最大陆地的面积           |
| **练习2+** | OJ 独立练习         | 举一反三               | P1451, P1596, P1605        |

**注意**：
- 练习1 必须在例题之前，用于巩固刚推导的核心逻辑
- 变式 在例题之后，是对例题的改动（如数块数→算面积）
- 练习2/3/4 是独立 OJ 题，与例题知识点相同但场景不同

## 语言风格

### 目标受众

14 岁中学生，调整内容匹配认知发展和兴趣。

### 风格要求

| 要求         | 说明                       |
| ------------ | -------------------------- |
| 机智幽默     | 用引人入胜、轻松的语言     |
| 适合年龄     | 不幼稚，也不过于学术化     |
| 贴近生活     | 游戏、社交媒体、日常生活   |
| 清晰但不简化 | 用精确术语，但可理解地解释 |
| 对话式语调   | 像和聪明青少年交谈         |

### 常用类比

| 概念                 | 类比                                           |
| -------------------- | ---------------------------------------------- |
| 变量                 | 贴标签的盒子                                   |
| 函数                 | 有输入输出的专用工具                           |
| 循环                 | 游戏中的重复动作                               |
| 数组                 | 编号的储物柜行                                 |
| 二维数组             | 教室座位表（第几排第几列）                     |
| 指针                 | 信封上的地址                                   |
| 递归                 | 俄罗斯套娃                                     |
| 栈                   | 盘子堆（后进先出）                             |
| 队列                 | 售票处排队（先进先出）                         |
| 二分查找             | 猜 1-100 数字，每次减半                        |
| 贪心算法             | 自助餐上总拿最大的那片披萨                     |
| 动态规划             | 记住数学题答案，不重新计算                     |
| DFS 搜索             | 探险家走迷宫：每条路走到黑，回头走另一条       |
| 连通块（Flood Fill） | 油漆桶填充：点一下就能填充整个区域             |
| 回溯                 | 走错了退回去换个方向重试                       |
| 方向数组             | 指南针模板：上(-1,0) 下(1,0) 左(0,-1) 右(0,1） |

## 讲义文件处理规则

更新讲义数据库时（`update_item.py`）：

1. **HTML 优先**：同时存在 `.html` 和 `.md`，只添加 HTML
2. **MD 过滤**：`.md` 文件必须以 `_ppt` 结尾才会被添加
   - `循环结构_ppt.md` ✅
   - `循环结构.md` ❌
3. **支持类型**：`.pdf`、`.html`、`.md`（需符合上述规则）

## 操作优化要点

基于实际备课经验，总结操作要点，避免常见错误。

### 📋 任务开始前确认清单

1. **明确具体操作**
   - **错误案例**：用户说"把七和八融合"，理解成"三和七"、"七和八"反复出错
   - **正确做法**：
     - 用户说"七和八" → 立即确认："您指的是章节七（数组比较）和章节八（直方图）吗？"
     - 用户说"合并a到b" → 立即确认："是将a中的XX章节合并到b中的XX位置吗？"
     - **不要猜测，不要自信，先确认再动手！**

2. **检查内容重复性**
   - **错误案例**：a中的"直方图"和b中的"趣味统计"都是桶计数法，内容重复
   - **正确做法**：
     - 合并前，先对比两个章节的知识点
     - 如果发现重复，立即询问用户："这两个章节内容相似，是删除还是合并？"
     - 避免重复内容出现在最终讲义中

3. **确认文件命名风格**
   - **错误案例**：自作主张重命名文件，但命名风格可能不符合用户期望
   - **正确做法**：
     - 询问用户："这个新讲义叫什么名字？要俏皮一点（如'巧用数组当桶使'）还是严肃一点（如'一维数组进阶应用'）？"
     - 查看现有文件的命名风格，保持一致性

### 🔧 操作执行阶段

4. **小步修改，实时验证**
   - **错误案例**：一次性大范围替换，结果替换错了内容
   - **正确做法**：
     - 每次只改一个章节（如只改章节编号）
     - 修改后，立即读取文件确认是否正确
     - 使用 TaskCreate 记录每一步，完成一项标记一项

5. **保留备份，防止误操作**
   - **正确做法**：
     - 重要修改前，先备份文件：`cp file.md file.md.bak`
     - 如果出错，可以立即恢复
     - 批量操作前，先在小范围测试

6. **实时与用户确认**
   - **正确做法**：
     - 每完成一个大步骤（如删除章节、重命名文件），询问用户："这样对吗？"
     - 不要等所有操作完成后才让用户检查

### ✅ 完成后检查阶段

7. **章节编号连续性检查**
   - **错误案例**：章节编号变成"一、三、四、五..."（跳过了二）
   - **正确做法**：
     - 修改章节后，立即检查编号是否连续
     - 子章节编号也要检查（1.1, 1.2, 1.3...）
     - 使用 Grep 工具查找所有章节标题，快速检查

8. **目录和链接同步更新**
   - **错误案例**：内容改了，但目录和 footer 中的链接还是旧的
   - **正确做法**：
     - 修改内容后，立即更新目录页（`CONTENTS`）
     - 更新 footer 中的页码链接（如 `[删除](#4)`）
     - 生成 PPT 后，验证所有链接是否正确跳转

9. **内容去重和完整性检查**
   - **正确做法**：
     - 检查是否有重复的知识点
     - 检查是否有冗余的示例
     - 确保每节课内容量适中（不要太少或太多）

### 🚨 备课常见错误与避免方法

| 错误类型           | 表现                            | 避免方法                                   |
| ------------------ | ------------------------------- | ------------------------------------------ |
| **理解错误**       | 把"七和八"理解成"三和七"        | 重复确认章节编号，不要自信猜测             |
| **错误添加内容**   | 从源文件复制了不该复制的内容    | 先明确"要什么"，而不是"不要什么"           |
| **编号混乱**       | 章节编号不连续（一、三、四...） | 每次添加/删除章节后，立即检查并修复编号    |
| **链接未同步**     | 目录和 footer 中的链接还是旧的  | 修改内容后，立即更新目录和 footer          |
| **内容重复**       | 相似知识点出现在多个章节        | 操作前先对比内容，合并重复部分             |
| **命名不符合期望** | 自作主张重命名，但风格不对      | 先询问用户命名风格，查看现有文件保持一致性 |
