## 教学设计步骤

### Step 1：用行为动词写目标

| ❌ 不好           | ✅ 好                         |
| ---------------- | ---------------------------- |
| "理解排序算法"   | "能手写冒泡排序和快速排序"   |
| "了解时间复杂度" | "能分析给定算法的时间复杂度" |
| "掌握 DFS"       | "能用 DFS 解决连通性问题"    |

### Step 2：选择教学策略

| 知识点类型 | 推荐策略       | 说明                                   |
| ---------- | -------------- | -------------------------------------- |
| 概念引入   | 探究式         | 先抛问题，让学生猜，再揭示             |
| 算法推导   | **问题驱动式** | **先抛具体问题，引导发现，再揭示模板** |
| 代码实现   | 项目式         | 给框架，学生填空                       |
| 综合应用   | 游戏化/竞赛    | 限时挑战、排行榜                       |

### Step 2b：问题驱动式模板推导法（核心新增）

**当教学内容涉及算法/模板推导时，必须使用此方法，禁止直接展示模板。**

#### 三步引导法

| 步骤                   | 做什么                         | 以 DFS 方向数组为例                                                                       |
| ---------------------- | ------------------------------ | ----------------------------------------------------------------------------------------- |
| **Step 1：抛具体问题** | 提出学生能感知的具体问题       | "DFS 要从一个格子出发，第一步该往哪走？" → 上/下/左/右 → 怎么用代码表示？→ 引导出方向数组 |
| **Step 2：给注释骨架** | 给出关键注释，让学生填核心代码 | `// 四个方向走一圈` → 学生自己写循环 + dfs 调用                                           |
| **Step 3：揭示模板**   | 作为推导的结论，而非起点       | "你看，这就是 DFS 搜索模板——我们从三个问题一步步推出来的"                                 |

#### 通用问题拆解框架

面对任何算法模板，拆解成 2-4 个具体问题让学生回答：

```
算法模板 → 拆解为 N 个具象问题 → 逐个引导发现 → 组合得出完整模板
```

示例：
| 算法     | 问题 1                 | 问题 2                         | 问题 3                  | 问题 4                 |
| -------- | ---------------------- | ------------------------------ | ----------------------- | ---------------------- |
| DFS 模板 | 往哪走？→ 方向数组     | 什么情况停下？→ 边界/障碍/回头 | 怎么防死循环？→ vis标记 | 怎么串起来？→ 递归调用 |
| 二分查找 | 从哪开始搜？→ 中间位置 | 怎么缩小区间？→ 比较大小       | 什么时候停？→ l<=r      | —                      |

### Step 3：设计情绪曲线（详细参见 [叙事设计](./叙事设计.md)）

波浪式上升：升→小降→升更高→小降→升到最高

```
导入（游戏场景）→ 2 分
  ↓
基础语法 → 5 分
  ↓ 小练习（小降）→ 4 分
核心技巧 → 7 分
  ↓ 挑战题（小降）→ 6 分
综合应用 → 9 分（最高点）
```

### Step 4：设计植入-揭示（详细参见 [叙事设计](./叙事设计.md)）

```
【植入】开场：为什么 N=1000 能过，N=10000 就超时？
    ↓ 空白期（讲其他内容，但有关联）
【揭示】因为你的算法是 O(n²)，改成 O(n log n) 就能过
```

### Step 5：设计赌注阶梯（详细参见 [叙事设计](./叙事设计.md)）

只升不降：基础→提高→挑战

### Step 6：规划评估方式

| 评估类型   | 时机 | 工具              |
| ---------- | ---- | ----------------- |
| 形成性评价 | 课中 | 随堂测（5-10 题） |
| 诊断性评价 | 课后 | OJ 提交数据       |
| 总结性评价 | 课末 | 作业完成情况      |

## 教育理论应用

| 理论 | 应用要点 | 对应环节 |
|------|---------|----------|
| 布鲁姆分类 | 目标分层：记→解→用→析→评→创 | 作业分层 |
| 最近发展区（ZPD） | 难度适中，动态调整 | 情绪曲线、赌注阶梯 |
| 通用学习设计（UDL） | 多模态：视→听→动 | 代码填空、图示、口述 |

## 线下角色设计（一对多，无助教）

| 角色 | 职责 | 对应内容 |
|------|------|----------|
| 主讲 | 独力授课，兼顾答疑、巡査 | 讲义主体、困惑点补充 |
| 活跃型生 | 善问，敢言 | 抢答环节 |
| 深思型生 | 追问，重理 | 深挖环节 |
| 实践型生 | 重应用，好动手 | 上台演示、分组互助 |

## 叙事设计（核心工具）

情绪曲线、植入-揭示分离、赌注阶梯的详细规则参见：
→ [叙事设计](./叙事设计.md)

**在 Design 阶段的应用要点**：
1. 画情绪曲线草图（标出高低点和转折位置）
2. 确定植入点和揭示点（中间留空白期）
3. 设计赌注阶梯（例题难度递进）
4. 安排互动环节位置（情绪小降点）

---

## 设计阶段检查清单

- [ ] 用行为动词写教学目标（"能实现"而不是"理解"）
- [ ] 设计情绪曲线（波浪式上升，画草图）
- [ ] 设计植入-揭示（哪里埋悬念、哪里揭示）
- [ ] 设计赌注阶梯（难度递进，只升不降）
- [ ] 规划评估方式（随堂测/作业/OJ 提交）
- [ ] 设计互动环节（提问、讨论、练习）
- [ ] 确定教学策略（讲授/探究/项目/游戏化）
- [ ] **对算法/模板类知识点，使用问题驱动式模板推导法**
- [ ] **拆解算法为 N 个具体问题，每个问题让学生能独立回答**
- [ ] **规划递进式练习序列（纸上练习→例题→变式→OJ练习）**
